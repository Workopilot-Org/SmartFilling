# v2 脚本配置操作手册

> 本手册面向脚本配置人员，帮助快速上手编写 v2 格式的智能填报脚本。所有配置字段均有详细说明、必填/选填标注和完整样例。

---

## 脚本功能概览

v2 脚本采用 **确定性执行 + AI 按需介入** 的混合架构：能用结构化指令描述的操作走确定性执行（零 token、毫秒级、100% 可复现），无法描述的复杂交互（弹窗搜索、自定义组件、验证码等）交给 AI 自主完成，确定性步骤失败时 AI 自动兜底修复。

> **与传统方案的对比优势**：传统 RPA 死板脆弱（选择器变了就挂），纯 AI 浏览器软件昂贵且不稳定（每步都消耗 token、结果不可复现）。v2 脚本取两者之长——确定性步骤保证速度和稳定性，AI 仅在需要时介入，兼具 RPA 的可靠性 + AI 的灵活性，同时将 token 消耗降低 90% 以上。

核心特性包括：

**22 种浏览器操作**
覆盖导航、点击、填写、选择、悬停、按键、滚动、上传、JS 执行、数据提取、截图、标签页管理等全部常见交互，录制器自动生成 XPath 选择器，手工编辑也可使用 CSS。

**三种执行模式**
- **sequential**（顺序）：按步骤依次执行，适用于登录、导航、表单填写等线性流程
- **loop**（循环）：按数据行循环执行，自动管理行索引，支持明细表、翻页等重复场景
- **ai**（AI 自主）：将整个阶段交给 AI 完成，适用于复杂弹窗搜索、非标准 UI 组件等无法用确定性步骤描述的场景

**AI 能力按需使用**
- **ai action**：单个步骤级 AI 调用，脚本编写时主动声明，适用于 layui 下拉框、复杂日期选择器、弹窗搜索选择等非标准交互
- **ai phase**：整个阶段由 AI 自主执行，多个子目标共享一个 AI 会话上下文，AI 灵活分配执行轮次
- **AI fallback**：确定性步骤失败时自动触发 AI 兜底修复，无需脚本额外配置（通过 `onError: "ai-fallback-stop"` 或 `"ai-fallback-skip"` 启用）
- **AI 视觉能力**：AI 可自主截图查看验证码、页面内容，用视觉能力识别和处理无法用代码描述的元素
- **验证码 AI 兜底**：captcha action 确定性识别失败后，AI 自动用视觉能力接管

**条件分支**
- **phase 级 condition**：phase 执行前检测条件（如填报数据中是否包含某个字段），不满足则跳过整个 phase，实现"有附件才上传"、"科目为差旅费才填写出差信息"等按数据分支
- **step 级 condition**：单个步骤执行前检测条件，不满足则跳过该步，比 phase 级更细粒度
- **check + detect**：步骤执行后进行条件检测，支持 20 种 detect 类型（URL 变化、元素可见/消失、对话框内容、页面文本、JS 表达式、数据字段是否存在等），配合 11 种 then 分支动作（nothing / continue / step_error / phase_success / phase_error / phase_rerun / row_rerun / script_success / script_fail / goto / break），实现保存后提取单据编号、提交失败自动清理、错误弹窗重试等全部业务分支逻辑
- **组合条件**：detect 支持 `all`（AND）、`any`（OR）、`not`（取反）嵌套组合，构建任意复杂条件

**灵活的流程控制**
- **goto**：跳转到任意 phase 或 step，支持跨层级跳转
- **phase 嵌套**：phase 的 steps 内可混合 step 和 phase 节点，实现条件分支嵌套在循环内、内层循环引用外层数据等复杂结构
- **循环控制**：loop phase 支持 continue（下一行）、break（退出循环）、row_rerun（重跑当前行）、phase_rerun（重跑整个 phase）、数据驱动 + 页面条件双重终止

**iframe 选择器链（string[]，根→叶）**

步骤/detect 通过 `iframe` 字段（**string[]，根→叶的 selector 链**，每层一个 XPath 或 CSS selector）定位到目标 iframe 内的元素，录制时自动反推生成，也可手写覆盖。详见各 action 的 `iframe` 字段说明与[第六节「iframe 选择器自动提取」](#iframe-选择器自动提取)。

**变量与数据提取**
- `{{fieldName}}` 占位符在运行时自动替换为填报数据
- `extract` action 提取页面文本、属性、URL、计数等数据
- `storeAs` 将提取结果存入变量，供后续步骤引用
- 作用域栈机制支持嵌套循环中的变量逐层查找

**字段嵌套**

`fields` 支持 `type: "array"` + `fields` 子数组递归嵌套，精确描述多层结构化数据。例如报销单→报销明细（科目级附件）→消费记录（行级附件）三层嵌套，每层可有独立的附件字段，与 loop phase 嵌套一一对应，作用域栈自动按层级查找数据。附件统一作为 `type: "file"` 字段嵌套在 fillData 对应位置，不再与填报数据分离。

**六层容错机制**

步骤条件跳过 → 重试 → 备选方案（fallback）→ 条件分支（check）→ AI 兜底（ai-fallback），通过 `onError` 五级配置链（step → phase → settings → appsettings → 默认值）精细控制错误策略。

**验证码处理**

内置 `captcha` action 支持 4 种验证码类型（文字识别、拼图滑块、像素差异、点选），确定性处理零 token 消耗，失败后 AI 视觉兜底。

**值变换**

字段级 `transform` 支持（trim、大小写），`format` 支持（日期格式转换、数字格式化，type=date/number 时自动转换），填入前自动应用。

**JSON Schema 编辑器支持**

v2 脚本提供完整的 JSON Schema 文件，在 VS Code 中引用后可获得智能补全、悬停提示、实时校验等能力，大幅降低编写门槛。详见[第十四节：JSON Schema 与编辑器使用](#十四json-schema-与编辑器使用)。

## 一、快速开始：最小可用脚本

以下是一个最简单的 v2 脚本骨架，包含登录一个系统并填写一个字段的完整流程：

```json
{
  "$schema": "script-v2.json",
  "version": 2,
  "scriptId": "my-first-script",
  "name": "示例脚本",

  "settings": {
    "defaultTimeout": 30000,
    "onError": "stop"
  },

  "phases": [
    {
      "kind": "phase",
      "name": "login",
      "aiGoal": "登录系统",
      "steps": [
        { "kind": "step", "action": "navigate", "url": "https://example.com/login" },
        { "kind": "step", "action": "fill", "selector": "//*[@id='username']", "value": "{{username}}" },
        { "kind": "step", "action": "fill", "selector": "//*[@id='password']", "value": "{{password}}" },
        { "kind": "step", "action": "click", "selector": "//*[@id='login-btn']" },
        { "kind": "step", "action": "wait", "ms": 2000 }
      ]
    },
    {
      "kind": "phase",
      "name": "fill_form",
      "aiGoal": "填写表单",
      "steps": [
        { "kind": "step", "action": "fill", "selector": "//*[@id='title']", "value": "{{title}}" },
        { "kind": "step", "action": "click", "selector": "//*[@id='submit']" }
      ]
    }
  ],

  "fields": [
    { "name": "username", "label": "用户名", "type": "string", "source": "system", "required": true },
    { "name": "password", "label": "密码", "type": "string", "source": "system", "required": true },
    { "name": "title", "label": "标题", "type": "string", "required": true }
  ]
}
```

**关键概念**：
- **phase**（阶段）：相关步骤的分组，按顺序执行
- **step**（步骤）：单个浏览器操作（点击、填写、等待等）
- **fields**（字段）：声明脚本需要哪些填报数据
- **`{{fieldName}}`**：占位符，运行时替换为实际填报数据

**注释支持**：脚本 JSON 文件支持 `//` 行注释与 `/* 块注释 */`，可在任意位置添加注解（标准 JSON 本不允许，引擎已放宽解析）：

```json
{
  "version": 2,
  // 顶部说明
  "scriptId": "my-first-script",
  "name": "示例脚本", /* 名称 */
  ...
}
```

> ⚠️ **注释会因脚本重新序列化而丢失**：脚本只要经过"对象→JSON"重新写回文件（典型路径：**重新录制同名脚本覆盖**、经 `/api/script/save` 提交），注释就会被丢弃——ScriptV2 对象模型不保留注释。前端目前**没有脚本内容编辑器**，脚本主要由录制器产出（产出本身不含注释）。**直接编辑 `data/scripts/*.json` 文件**所加的注释会一直保留，直到该脚本被重新录制或重新保存覆盖。若需说明**长期保留**（不受重新录制影响），建议写进各对象的描述字段——脚本顶层 / phase 的 `aiGoal` 与 `description`、step 与 field 的 `description`——这些字段随对象一起序列化保存，且 step 的 `description` 还会作为意图提示传给 AI fallback。

## 二、脚本顶层结构

```json
{
  "$schema": "script-v2.json",
  "version": 2,
  "scriptId": "唯一标识",
  "documentTypeId": "单据类型ID",
  "name": "脚本名称",
  "description": "脚本描述",
  "settings": { ... },
  "phases": [ ... ],
  "fields": [ ... ]
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `$schema` | string | 否 | JSON Schema 文件路径，用于 VS Code 智能提示和校验。如 `"script-v2.json"` |
| `version` | int | **是** | 脚本版本号，固定为 `2` |
| `scriptId` | string | **是** | 脚本唯一标识，如 `"f539de01"` |
| `documentTypeId` | string | 否 | 关联的单据类型 ID |
| `name` | string | **是** | 脚本名称，如 `"HK销售发票"` |
| `description` | string | 否 | 脚本描述，说明脚本用途 |
| `settings` | object | 否 | 脚本业务配置，覆盖引擎运行时默认值（[详见第三节](#三settings-脚本业务配置)） |
| `phases` | array | **是** | 执行阶段列表（[详见第五节](#五phases分阶段执行)） |
| `fields` | array | 否 | 填报字段定义列表（[详见第四节](#四fields字段定义)） |

## 三、settings 脚本业务配置

`settings` 用于覆盖引擎运行时配置（appsettings.json），但**不能修改纯运行时配置**（headless、slowMo、debugMode、browserPath 等）。

```json
{
  "settings": {
    "defaultTimeout": 30000,
    "defaultAiTimeout": 180000,
    "stepRetry": { "count": 2, "interval": 1000 },
    "onError": "ai-fallback-stop",
    "viewport": { "width": 1600, "height": 900 },
    "maxAiTurns": 30,
    "screenshot": { "onStepFailure": true },
    "maxScriptDuration": 300000
  }
}
```

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `defaultTimeout` | int | 30000 | 本脚本默认超时（毫秒），覆盖 appsettings |
| `defaultAiTimeout` | int | 180000 | **AI 兜底（step 级 AiFallback）/ phase 级 AI fallback 的独立超时（毫秒）**，覆盖 appsettings，与确定性步骤超时分离。AI 多轮调用耗时较长，用独立预算避免误超时。确定性步骤**不**消耗此预算 |
| `stepRetry` | object | `{count:1, interval:1000}` | 本脚本步骤重试配置 |
| `stepRetry.count` | int | 1 | 重试次数（0 = 不重试） |
| `stepRetry.interval` | int | 1000 | 重试间隔（毫秒） |
| `onError` | string | `"stop"` | 本脚本全局错误策略。可选值：`stop` / `skip` / `ai-fallback-stop` / `ai-fallback-skip` |
| `viewport` | object | 1280x720 | 本脚本需要的视口大小 |
| `viewport.width` | int | 1280 | 视口宽度 |
| `viewport.height` | int | 720 | 视口高度 |
| `maxAiTurns` | int | 30 | 本脚本 AI 最大执行轮次 |
| `screenshot` | object | 全 false | 本脚本截图配置 |
| `screenshot.onPhaseProgress` | bool | false | 每步执行后截图并发送 |
| `screenshot.onStepFailure` | bool | false | 步骤失败时保存截图到文件 |
| `screenshot.onTaskComplete` | bool | false | 任务成功时截图附带完成消息 |
| `screenshot.onTaskFailure` | bool | false | 任务失败时截图附带失败消息 |
| `screenshot.compressQuality` | int | 70 | 截图压缩质量。0或>=100不压缩(PNG)，1-99用JPEG+该quality |
| `screenshot.actionDefaultFolder` | string | 配置值 | screenshot action 的默认保存目录。支持三级覆盖：step.folder → settings.screenshot.actionDefaultFolder → appsettings Screenshot.ActionDefaultFolder |
| `screenshot.onStepFailureFolder` | string | `"logs/"` | 步骤失败截图的保存目录（仅全局配置，不分级覆盖）。默认 "logs/" |
| `maxScriptDuration` | int | 600000 | 本脚本执行时间上限（毫秒） |

### 配置优先级

```
step 级（timeout / retry / onError / maxAiTurns）
  ↓ 覆盖
phase 级（onError / maxAiTurns）
  ↓ 覆盖
script.settings（脚本业务配置）
  ↓ 覆盖
appsettings.json（运行时全局默认）
```

## 四、fields：字段定义

> **（形态 A 前写法，已废弃）** 旧版本曾用顶层 `iframeConfig` 集中声明所有 iframe 帧（`frames[]` 树结构），步骤中通过 `iframe: "frameId"` 字符串引用。**形态 A（iframe selector 链固化）改造后已移除**：`iframe` 字段改为 **string[] selector 链（根→叶）**，录制时自动反推生成、可直接手写覆盖，不再需要 `iframeConfig` 预声明。旧脚本（含 `iframeConfig` / 字符串 `iframe`）**不兼容、不迁移**，需重新录制。iframe 字段语义详见各 action 的 `iframe` 字段说明、[第五节 phase 级 iframe](#phase-字段说明)、[第六节 iframe 选择器自动提取](#iframe-选择器自动提取) 与附录速查表。


`fields` 定义脚本涉及的填报字段。用途：对话收集数据、AI fallback 理解上下文、可视化编辑器渲染表单。

### 字段属性全览

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|:----:|--------|------|
| `name` | string | **是** | - | 字段名，对应 fillData 中的 key 和步骤中的 `{{name}}`。**脚本内全局唯一** |
| `label` | string | **是** | - | 显示名称，如"合同名称"、"签订日期" |
| `type` | string | **是** | `"string"` | 数据类型：`string` / `number` / `date` / `boolean` / `file` / `array` |
| `uiComponent` | string | 否 | `"input"` | UI 组件类型（见下方 uiComponent 表） |
| `source` | string | 否 | null | 数据来源：`user`（询问用户）/ `system`（系统变量，由调用方提供，引擎不校验完整性）/ `computed`（计算得出，引擎收集层排除、不询问用户）。默认 null（运行时视作 user）。**DEC-7 硬约束**：字段名为 `username`/`password`（系统凭据字段，Worker fillData 硬编码这两个 key 存 payload 用户名/解密密码）时，`source` 必须 `system`（否则 `LoadFromJson` 报错——其他 source 会被 Worker 硬编码值覆盖，`{{username}}`/`{{password}}` 取错值，silent） |
| `required` | bool | 否 | false | 是否必填 |
| `defaultValue` | any | 否 | null | 默认值（仅声明/兜底，回放不合并进 fillData） |
| `options` | string[] | 否 | null | 可选项列表（select/radio 时使用） |
| `format` | string | 否 | null | 格式约束：前端展示格式 + 回放时 date/number 自动转换（date: `"YYYY-MM-DD"`、number: `"#,##0.00"`） |
| `min` / `max` | number | 否 | null | 数值/日期范围限制 |
| `maxLength` | int | 否 | null | 最大长度限制（string 类型） |
| `pattern` | string | 否 | null | 正则校验（string 类型），如 `"^1\d{10}$"` |
| `description` | string | 否 | null | 字段说明，帮助 AI 理解字段含义和填写规则 |
| `placeholder` | string | 否 | null | 输入提示文本 |
| `multiple` | bool | 否 | false | 是否多选/多文件 |
| `items` | object | 否 | null | 简单值数组元素定义（见 items 说明） |
| `fields` | FieldDef[] | 否 | null | 嵌套子字段定义（结构化行时使用，见嵌套字段） |
| `transform` | string | 否 | null | 填入前的值变换规则（见 transform 表） |

### type（数据类型）

| type | 说明 | 校验规则 |
|------|------|---------|
| `string` | 字符串 | maxLength, pattern |
| `number` | 数字 | min, max, format |
| `date` | 日期 | format（默认 `"YYYY-MM-DD"`），min, max |
| `boolean` | 布尔值（checkbox/radio 字段用此类型） | - |
| `file` | 文件附件。fillData 中存为 `[{name, url, path}]` | `multiple` 控制单/多文件 |
| `array` | 数组。有 `fields` 时为结构化行，有 `items` 时为简单值列表 | - |

### uiComponent（UI 组件类型）

| uiComponent | 说明 | 典型关联 |
|-------------|------|---------|
| `input` | 文本输入框 | type: string/number |
| `textarea` | 多行文本 | type: string |
| `select` | 下拉选择 | options, multiple |
| `radio` | 单选按钮组 | options |
| `checkbox` | 复选框 | options, multiple |
| `upload` | 文件上传 | type: file, multiple |
| `click-choose` | 点击弹窗选择（如搜索选择供应商） | description 中说明选择方式 |
| `datepicker` | 日期选择器 | type: date, format |
| `table` | 表格（数组+结构化列） | type: array + fields |
| `hidden` | 隐藏字段 | 不在 UI 显示 |

### source（数据来源）

| source | 说明 | 典型字段 |
|--------|------|---------|
| `user` | 对话时询问用户（默认） | 合同名称、金额、日期 |
| `system` | 系统变量，不询问用户（由调用方（前端输入框/WS payload）提供，引擎不校验完整性（决策5a），不通过 AI 对话询问） | username、password |
| DEC-7 硬约束 | **加载期硬校验（`ScriptLoader.LoadFromJson` 拒绝）**：① 字段名为 `username`/`password`（系统凭据字段，Worker fillData 硬编码这两个 key 存 payload 用户名/解密密码）时 `source` 必须 `system`（其他 source 会被 Worker 硬编码值覆盖→`{{username}}`/`{{password}}` 取错值，silent）；② `storeAs` 变量名不能用 `username`/`password`（会被 fillData 系统凭据在 scopeChain 末层永久遮蔽取错值，silent）。手写脚本违反上述任一会被 `LoadFromJson` 拒绝；录制期提示词（RecordingPrompt 命名规则）+ 录制层实时反馈（RecordingEngine）软引导前置 |
| `computed` | 引擎或脚本计算得出 | 总金额、billCode |

### items（简单值数组元素定义）

`type: "array"` 且无 `fields` 时使用，描述数组元素结构：

```json
{ "name": "tags", "type": "array", "items": { "type": "string", "options": ["紧急", "需跟进", "已审核"] } }
```

| items 字段 | 类型 | 说明 |
|-----------|------|------|
| `type` | string | 元素数据类型：string / number / date / boolean / file |
| `options` | string[] | 可选值列表 |
| `min` / `max` | number | 数值/日期范围 |
| `format` | string | 格式约束 |

### 嵌套字段（结构化行）

`type: "array"` 且有 `fields` 子数组时，定义结构化表格行。`fields` 可递归嵌套：

```json
{
  "name": "EXPENSE_RECORDS", "label": "报销明细", "type": "array", "uiComponent": "table",
  "fields": [
    { "name": "category", "label": "报销科目", "type": "string", "uiComponent": "select",
      "options": ["差旅费", "办公费用", "招待费"] },
    {
      "name": "consumption", "label": "消费记录", "type": "array", "uiComponent": "table",
      "fields": [
        { "name": "consumeDate", "label": "消费日期", "type": "date" },
        { "name": "consumeAmount", "label": "消费金额", "type": "number" },
        { "name": "consumeAttachments", "label": "消费附件", "type": "file", "multiple": true }
      ]
    }
  ]
}
```

**区分**：`fields` 存在 → 结构化行（表格列定义）；`items` 存在 → 简单值列表。

### transform（值变换）

填入表单前对值自动进行文本变换（仅 trim/大小写，**加载期 schema enum 校验**——非法值如 `"Trim"`/`"trim "`（大小写/尾空格不精确匹配）会被拒）。**日期/数字格式转换由 `format` 承担**（见字段 `format`）：

| transform | 说明 | 示例 |
|-----------|------|------|
| `"trim"` | 去除首尾空白 | `" hello "` → `"hello"` |
| `"upper"` | 转大写 | `"abc"` → `"ABC"` |
| `"lower"` | 转小写 | `"ABC"` → `"abc"` |

### 字段定义完整示例

```json
{
  "fields": [
    { "name": "username", "label": "用户名", "type": "string", "uiComponent": "input", "source": "system", "required": true },
    { "name": "F_HTMC", "label": "合同名称", "type": "string", "uiComponent": "input", "required": true, "maxLength": 200 },
    { "name": "F_HTFL", "label": "合同分类", "type": "string", "uiComponent": "select", "required": true,
      "options": ["劳务分包合同", "材料采购合同-直购", "租赁分包"] },
    { "name": "F_QDRQ", "label": "签订日期", "type": "date", "uiComponent": "datepicker", "required": true },
    { "name": "F_HTQDJE", "label": "合同金额", "type": "number", "uiComponent": "input", "min": 0, "format": "#,##0.00" },
    { "name": "F_YiFang", "label": "乙方", "type": "string", "uiComponent": "click-choose", "required": true,
      "description": "点击输入框后弹窗搜索选择供应商" },
    { "name": "TRANDATE", "label": "日期", "type": "date", "format": "DD/MM/YYYY" },
    { "name": "attachments", "label": "附件", "type": "file", "uiComponent": "upload", "multiple": true },
    { "name": "tags", "label": "标签", "type": "array", "items": { "type": "string", "options": ["紧急", "需跟进"] } },
    {
      "name": "ORDER_DETAILS", "label": "货品明细", "type": "array", "uiComponent": "table",
      "fields": [
        { "name": "PRODUCT", "label": "货品", "type": "string", "required": true },
        { "name": "QUANTITY", "label": "数量", "type": "number", "min": 0 },
        { "name": "PRICE", "label": "价格", "type": "number" }
      ]
    }
  ]
}
```

> **date 字段 format 写法**（F3）：date 的 `format` 同时支持 **moment.js 风格**（`YYYY-MM-DD` / `DD/MM/YYYY` / `ddd`=Mon / `dddd`=Monday）与 **C# 风格**（`yyyy-MM-dd`），两种皆可——引擎内置 token 规范化（`YYYY`→`yyyy`、`DD`→`dd`，大小写敏感、按长度降序），统一正确输出。number 的 `#,##0.00` 两套通用。
>
> 示例（设日期为 2026-06-23，星期一）：
>
> | format | 输出 | 说明 |
> |--------|------|------|
> | `YYYY-MM-DD` | `2026-06-23` | moment 风格，年-月-日 |
> | `yyyy-MM-dd` | `2026-06-23` | C# 风格，等价 |
> | `DD/MM/YYYY` | `23/06/2026` | moment 风格，日/月/年 |
> | `ddd` | `Mon` | 星期缩写（moment/C# 一致）|
> | `dddd` | `Monday` | 星期全名（moment/C# 一致）|
>
> **不支持**（C# 无等价 token，需改用 `evaluate` action）：星期几数字（moment `d`=0-6）、序数型（`Do`/`Mo`/`Qo`）、极短缩写（`dd`=Mo/Tu）。星期几数字示例：
>
> ```json
> { "action": "evaluate", "code": "new Date().getDay()", "storeAs": "weekDayNum" }
> ```

## 五、phases：分阶段执行

`phases` 是脚本的核心，定义了浏览器操作的执行流程。每个 phase 是一组相关步骤的集合，支持三种执行模式。

### 统一节点模型

phases 数组中的每个元素通过 `kind` 字段标识类型：

| kind | 类型 | 说明 |
|------|------|------|
| `"phase"` | phase 节点 | 一组步骤的集合，有 name、type、steps 等字段 |
| `"step"` | step 节点 | 单个浏览器操作（仅用在 phase 的 steps 数组内部） |

> **注意**：顶层 `phases` 数组中只能放 phase 节点，不能放 step 节点。

### phase 字段说明

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|:----:|--------|------|
| `kind` | string | **是** | - | 固定为 `"phase"` |
| `name` | string | **是** | - | phase 名称，**脚本内全局唯一**（goto 跳转用） |
| `type` | string | 否 | `"sequential"` | 执行类型：`sequential`（顺序）/ `loop`（循环）/ `ai`（AI 自主执行） |
| `aiGoal` | string | 是 | - | phase 的业务目标描述，**R3-2 必填**（ScriptLoader 校验缺失即加载失败；录制完成后自动生成，无需用户填。取消/超时/AI 未回填的不完整脚本，保存时可在弹框确认后**强制保存**跳过校验，脚本列表标 ⚠️；Worker 填报执行会报失败，不会 silent） |
| ~~`relevantFields`~~ | ~~string[]~~ | ~~否~~ | - | ~~已废弃~~。引擎自动从步骤中提取字段名，无需手动指定 |
| `condition` | object | 否 | - | phase 执行前条件，不满足则跳过（直接写 detect 对象） |
| `onError` | string | 否 | 继承上级 | 覆盖全局错误策略 |
| `iframe` | string[] | 否 | - | phase 级默认 iframe **selector 链（根→叶，每层一个 XPath 或 CSS selector）**，本 phase 内未指定 `iframe` 的 step/detect 继承此值。`null`/省略=未配（继承上层）；`[]`（显式空数组）=强制主文档（不继承、覆盖回主文档）；有内容=指定链。一般由录制自动反推生成，也可手写覆盖。示例：`["//iframe[@id='form']"]`、嵌套 `["//iframe[@id='main']","//iframe[@name='inner']"]` |
| `steps` | array | **是** | - | 子节点列表（step 和 phase 可混合嵌套） |
| **loop 专用** | | | | |
| `loopSource` | string | loop | - | 循环数据源字段名，从作用域栈逐层查找对应的数组。**写裸字段名（不写 `{{}}`，如 `items` 而非 `{{items}}`），必须是 fields 中 `type=array` 的字段**（明细表，支持嵌套 rowData）；不支持 vars。简单值数组（`array+items`）loop 内用 `{{item}}` 引用当前标量；结构化行用 `{{字段名}}` |
| `loopCondition` | object | 否 | - | 按页面条件终止循环（每次迭代开始前检测） |
| `maxLoopCount` | int | 否 | 100 | phase 最大允许迭代数，**超过即失败**（loopSource 应配 `≥ 预期行数`，rows 完即正常成功终止；loopCondition 达上限=死循环超限失败）。正确配置不会达上限 |
| `rowIndexOffset` | int | 否 | 0 | 行索引偏移。目标系统从 1 开始时设 `1` |
| **ai phase 专用** | | | | |
| `maxAiTurns` | int | 否 | 30 | AI 最大执行轮次 |
| `timeout` | int | 否 | 继承 settings | phase 执行超时（毫秒），超时按 onError 处理 |

> **ai phase 步骤限制**：ai phase 的 steps 中只允许 action 必传为 `"ai"`（不可省略，决策 B 取消可省略）的 StepNode，不允许嵌套子 PhaseNode。

### 名称唯一性规则

| 名称类型 | 唯一性范围 | 原因 |
|---------|-----------|------|
| field name | 脚本内全局唯一 | 变量替换按作用域栈从内到外查找 |
| phase name | 脚本内全局唯一 | goto toPhase 需要在嵌套树中精确定位 |
| step name | 同一父 phase 的 steps 内唯一 | goto toStep 只在当前 phase 的直接 steps 中查找 |

step name 必传（每个 step 都要有 name，同 phase 内唯一，goto toStep 引用目标）。决策13/审计 #42：与录制端 tool schema required name + script-v2.json stepNode required name 全链路对称。

### sequential phase（顺序执行，默认）

```json
{
  "kind": "phase", "name": "login",
  "aiGoal": "登录系统",
  "steps": [
    { "kind": "step", "action": "fill", "selector": "//*[@name='INPUT_USERID']", "value": "{{username}}" },
    { "kind": "step", "action": "fill", "selector": "//*[@name='PASSWORD']", "value": "{{password}}" },
    { "kind": "step", "action": "click", "selector": "//*[normalize-space(.)='登入 / Login']" },
    { "kind": "step", "action": "wait", "ms": 2000 },
    { "kind": "step", "action": "check", "detect": { "type": "url_changed" }, "then": "continue" }
  ]
}
```

### loop phase（循环阶段）

用于明细表、翻页等重复执行的步骤：

```json
{
  "kind": "phase", "name": "fill_detail",
  "type": "loop",
  "loopSource": "ORDER_DETAILS",
  "aiGoal": "录入货品明细",
  "iframe": ["//iframe[@name='bottomframe']"],
  "steps": [
    { "kind": "step", "action": "fill", "selector": "//*[@name='PRODUCT']", "value": "{{PRODUCT}}", "pressEnter": true },
    { "kind": "step", "action": "fill", "selector": "//*[@name='QUANTITY']", "value": "{{QUANTITY}}" },
    { "kind": "step", "action": "fill", "selector": "//*[@id='PRICE']", "value": "{{PRICE}}", "skipIfDataEmpty": true },
    { "kind": "step", "action": "click", "selector": "//*[@type='submit']", "preSetup": { "dialogHandler": "auto_accept" } },
    { "kind": "step", "action": "check", "detect": { "type": "new_row_appears", "selector": "#data-table tbody tr" }, "then": "continue" }
  ]
}
```

**循环终止条件**：

| 字段 | 说明 |
|------|------|
| `loopSource` | 遍历数据源数组（fields 中 type=array 字段），rows 完即正常终止（成功）；写裸字段名不写 `{{}}` |
| `loopCondition` | 每次迭代开始前检测页面条件，不满足则终止。⚠️ 语义=页面条件：检查**行内元素**时需手写 `{{rowIndex}}`（录制不自动参数化 phase 级条件，详见上方「phase 级字段的变量支持」）；**不能**用行字段（`{{字段名}}`）——loopCondition 检测时机在迭代取行之前，行数据未注入；loopSource 控制行数 |

两者可共存。

**循环中的特殊变量**：
- `{{rowIndex}}`：当前行索引（从 0 开始），加上 `rowIndexOffset` 后为实际值

### loop 行级失败处理（业务失败 → 行级 AI）

loop 中**业务失败**（如保存报错、校验失败——click 成功点了但业务没过）与**步骤抛异常**（元素找不到/超时）处理路径不同：

- **步骤抛异常** → step retry/fallback → `#23` return PhaseResult(false) → phase onError（自动路径）
- **业务失败**（click 不抛异常）→ 需用 **check 检测失败 → `then: "goto"` 一个 `ai` action step 只修当前行**（办法 C）

**优点**：业务失败用主动 `ai` action 声明「这步用 AI」语义自然；`ai` action **不受 step AI fallback 的越界护栏（`#52`，仅 step 级专属）约束**，可自由回溯修 + 重存当前行；用现成 detect（`dialog_contains` 等），无需写检测 JS；不改引擎。

**样例**（loop 行内：保存 → 检测报错弹窗 → goto 行级 AI 修当前行 → 检测新行 → 继续）：

```json
{ "kind": "phase", "name": "fill_detail", "type": "loop", "loopSource": "ORDER_DETAILS", "steps": [
  { "kind": "step", "action": "click", "selector": "//*[@id='save-row-btn']" },
  { "kind": "step", "action": "check",
    "detect": { "type": "dialog_contains", "keywords": ["保存失败", "必填", "库存不足"] },
    "then": "goto", "toStep": "row-ai-fix" },
  { "kind": "step", "action": "check",
    "detect": { "type": "new_row_appears", "selector": "#data-table tbody tr" }, "then": "continue" },
  { "kind": "step", "name": "row-ai-fix", "action": "ai",
    "description": "当前行保存失败。诊断页面报错，结合行数据（货品={{PRODUCT}},数量={{QUANTITY}}）修正（补值/关弹窗/换方式）后重新点击保存，直到该行成功",
    "field": "PRODUCT", "maxAiTurns": 8 }
]}
```

> 三类 loop 失败处理互补：**步骤抛异常** → `#23`（return PhaseResult(false) → phase onError）；**业务失败** → 本模式（check → goto ai action）；**check 耗尽 rerun 次数** → `row_rerun` / `phase_rerun`。

### ai phase（AI 自主执行阶段）

将整个阶段交给 AI 自主完成，所有步骤合并为一个 AI 会话共享上下文：

```json
{
  "kind": "phase", "name": "search_supplier",
  "type": "ai",
  "maxAiTurns": 20,
  "iframe": ["//iframe[@name='layui-layer-iframe1']"],
  "steps": [
    { "kind": "step", "description": "搜索供应商", "field": "F_YiFang", "value": "{{F_YiFang}}" },
    { "kind": "step", "description": "从结果中选择第一个供应商", "selector": "//*[contains(@class,'search-results')]" },
    { "kind": "step", "description": "点击确认按钮", "selector": "//*[@id='confirm-btn']" }
  ]
}
```

ai phase 中的 step 只需 `description`（必填），`selector`/`field`/`value`/`storeAs` 作为可选提示。**action 必传为 `"ai"`（不可省略，决策 B 取消可省略）**，不允许其他 action 类型。**不允许嵌套子 phase**（steps 中只能放 step 节点，不能放 phase 节点）。

### phase 级 condition

phase 可设置执行前条件，不满足则跳过：

```json
// 检查 fillData 中是否有附件
"condition": { "type": "data_exists", "field": "attachments" }

// JS 表达式条件（服务端执行，可访问 fillData 和 vars）
"condition": { "type": "js", "check": "fillData.amount > 0" }

// 组合条件
"condition": { "all": [
  { "type": "data_exists", "field": "attachments" },
  { "not": { "type": "data_exists", "field": "skipAttachments" } }
]}
```

### phase 级字段的变量支持（`{{}}`）

`phase.iframe`（selector 链每层）、`phase.condition` 内的 `selector`/`keywords`/`value`、`phase.loopCondition` 内同款字段，均支持 `{{变量}}`/`{{字段名}}`/`{{rowIndex}}` 占位符，回放时自动替换（与 step 级一致）。

> **loop 边界提示**：`phase.loopCondition` 若检查**行内元素**（如每行某控件是否存在），其 `selector` 的行索引需**手写 `{{rowIndex}}`**（录制不自动参数化 phase 级条件）；`phase.condition`（phase 执行前一次评估，时机在取行之前）通常不涉及行索引。

### phase 嵌套

phase 的 steps 数组中可以混合 step 和 phase 节点，实现条件分支、内层循环等复杂结构：

```json
{
  "kind": "phase", "name": "fill_expenses", "type": "loop", "loopSource": "EXPENSE_RECORDS",
  "steps": [
    { "kind": "step", "action": "click", "selector": "//*[@id='add-expense']" },
    { "kind": "step", "action": "select", "selector": "//*[@id='category']", "value": "{{category}}" },
    {
      "kind": "phase", "name": "fill_travel",
      "condition": { "type": "js", "check": "fillData.category==='差旅费'" },
      "steps": [
        { "kind": "step", "action": "fill", "selector": "//*[@id='location']", "value": "{{travelLocation}}" },
        { "kind": "step", "action": "fill", "selector": "//*[@id='travelDate']", "value": "{{travelDate}}" }
      ]
    },
    {
      "kind": "phase", "name": "fill_office",
      "condition": { "type": "js", "check": "fillData.category==='办公费用'" },
      "steps": [
        { "kind": "step", "action": "fill", "selector": "//*[@id='feeDesc']", "value": "{{feeDesc}}" },
        { "kind": "step", "action": "fill", "selector": "//*[@id='feeAmount']", "value": "{{feeAmount}}" }
      ]
    }
  ]
}
```

## 六、selector 选择器使用指南

selector 字段用于定位页面上的目标元素。v2 引擎基于 Playwright，录制器自动生成 XPath 选择器，手工编辑时也可使用 CSS 选择器。

> **⚠️ 字符要求（加载期校验）**：selector 必须使用 **ASCII 直引号**（`'` `"`）。中文弯引号（`‘` `’` `“` `”`，U+2018/2019/201C/201D）与全角空格（`　`，U+3000）会被加载期 `ValidateSelectorCharset` 拒绝（手工编辑/中文输入法易误带入，否则回放才崩 Playwright `document.evaluate` 报 not a valid XPath expression）。中文逗号 `，`/括号 `（）`/冒号 `：` 不校验（在 `:has-text('中文，文本')` 等参数里合法）。校验覆盖 step/detect/fallback/captcha 选择器与 iframe 链各层。

### 支持的语法类型

| 语法 | 写法 | 说明 |
|------|------|------|
| XPath | `//*[@id='submit']`、`//input[@name='phone']` | 录制器自动生成的格式，推荐 |
| CSS 选择器 | `#id`、`.class`、`[attr='value']` | 手工编辑时可用，引擎自动识别 |
| 正则表达式 | `text=/pattern/flags` | 文本模式匹配，处理动态内容 |

注意：`text=/pattern/flags` 是 Playwright 私有正则语法，录制器不会生成此格式。仅用于手工编辑处理动态文本场景。XPath 1.0 不支持正则表达式。

### 匹配方式

#### 等于（精确匹配）

```json
{ "action": "click", "selector": "//*[@id='submitBtn']" }
{ "action": "click", "selector": "//*[@name='username']" }
{ "action": "click", "selector": "//*[@title='Submit']" }
```

| 语法 | 说明 |
|------|------|
| `//*[@id='value']` | 按 ID 精确匹配 |
| `//*[@attr='value']` | 按属性值精确匹配 |
| `//*[normalize-space(.)='text']` | 按文本精确匹配 |

#### 包含（部分匹配）

```json
{ "action": "click", "selector": "//*[contains(@title,'Sub')]" }
{ "action": "click", "selector": "//*[contains(@src,'/api/submit')]" }
```

| 语法 | 说明 |
|------|------|
| `//*[contains(@attr,'value')]` | 属性值包含指定字符串 |
| `//*[starts-with(@attr,'value')]` | 属性值以指定字符串开头 |
| `//*[contains(text(),'value')]` | 文本包含指定字符串 |

#### AND（多条件组合）

多个条件写在同一个 selector 中，全部满足才匹配：

```json
{ "action": "click", "selector": "//*[contains(@title,'Submit') and contains(@class,'btn-primary')]" }
{ "action": "fill", "selector": "//div[@id='form']//input[@type='text']" }
```

| 语法 | 说明 |
|------|------|
| `//*[cond1 and cond2]` | 多属性 AND 组合 |
| `//parent//child[@attr]` | 祖先后代层级组合 |

#### OR（多选一，用 fallback 链）

当同一个操作在不同情况下需要不同的 selector 时，使用 `fallback` 链依次尝试：

```json
{
  "action": "click",
  "selector": "//*[@id='submitBtn']",
  "fallback": {
    "selector": "//*[@title='Submit']",
    "fallback": { "selector": "//*[normalize-space(.)='提交']" }
  }
}
```

执行顺序：先试 `//*[@id='submitBtn']` → 失败试 `//*[@title='Submit']` → 再失败试 `//*[normalize-space(.)='提交']` → 最终走 `onError` 策略。

> fallback 省略 `action` 则**继承 step 的 action**（#21：原文档说"默认 click"与代码矛盾，实际 `fallback.Action ?? step.Action`）。显式写可覆盖，如填写类 step 的 fallback 想换成点击需显式 `"action": "click"`。

```json
{
  "action": "fill",
  "selector": "//*[@id='date']",
  "value": "2026-01-01",
  "fallback": { "action": "fill", "selector": "//*[@name='date']", "value": "2026-01-01" }
}
```

#### 正则表达式（文本模式匹配）

```json
{ "action": "click", "selector": "text=/提交|Submit|送信/" }
{ "action": "click", "selector": "text=/submit/i" }
{ "action": "click", "selector": "//button[matches(@title, 'Submit|提交', 'i')]" }
```

| 语法 | 说明 | 适用场景 |
|------|------|---------|
| `text=/pattern/` | 文本正则匹配（区分大小写） | 文本中含动态数字、日期等 |
| `text=/pattern/i` | 文本正则匹配（不区分大小写） | 大小写不确定的按钮文本 |
| `//...[matches(@attr, 'pattern', 'flags')]` | XPath 属性正则匹配 | 属性值含动态部分 |

**正则适用场景**：

| 场景 | selector | 说明 |
|------|---------|------|
| 分页文本 "共5条" | `text=/共\d+条/` | 数字会变化 |
| 进度 "Loading 75%" | `text=/Loading \d+%/` | 百分比会变化 |
| 订单号 "订单#12345" | `text=/订单#\d+/` | 编号会变化 |
| 多语言按钮 | `text=/提交\|Submit\|送信/` | 覆盖多种语言 |
| 日期文本 | `text=/\d{4}-\d{2}-\d{2}/` | 日期格式 |

### selector 中的变量替换

selector 字段支持 `{{}}` 占位符，运行时替换为实际值：

```json
{ "action": "click", "selector": "//*[@title='{{btnTitle}}']" }
{ "action": "click", "selector": "text={{menuName}}" }
{ "action": "click", "selector": "text=/{{btnPattern}}/" }
```

在 `fields` 中声明对应字段，由系统注入值：

```json
{ "name": "btnTitle", "label": "按钮标题", "type": "string", "source": "system" }
```

### 多语言场景处理

当页面元素属性（如 title、text）随语言变化时，有两种处理方式：

**方式一：`{{}}` 参数化**（推荐）

```json
{
  "action": "click",
  "selector": "//*[@title='{{submitBtnTitle}}']"
}
```

不同语言传入不同值：`{ "submitBtnTitle": "提交" }` 或 `{ "submitBtnTitle": "Submit" }`。

**方式二：正则覆盖**

```json
{ "action": "click", "selector": "text=/提交|Submit|送信/" }
```

一个 selector 覆盖所有语言变体，无需额外字段。

### URL 属性处理

当元素的 `src`、`href` 等属性包含动态查询参数时，使用部分匹配只取路径部分：

```json
// 验证码图片 src="/api/captcha?t=1234567890"，参数每次变化
{ "action": "screenshot", "selector": "//img[contains(@src,'/api/captcha')]" }

// 链接 href="/page?id=123&token=abc"，参数每次变化
{ "action": "click", "selector": "//a[contains(@href,'/page')]" }
```

不要使用完整 URL 精确匹配：
```json
// 错误：查询参数会变
{ "action": "click", "selector": "//a[@href='/page?id=123&token=abc']" }
```

### 录制时的 selector 自动提取

录制引擎按以下优先级自动提取 selector：

| 优先级 | 方式 | 稳定性 | 说明 |
|:---:|------|:---:|------|
| 1 | id / name / data-* | 高 | 精确匹配，自动跳过动态值 |
| 2 | 多属性组合（如 @name and @type） | 高 | 单属性不唯一时自动组合 |
| 3 | 祖先路径（属性锚点 + 文本锚点） | 中高 | 找到有特征的祖先，拼接相对路径 |
| 4 | 近邻锚点（旁边元素的文本或属性） | 中高 | 找到附近的文本标签或属性特征 |
| 5 | 属性包含匹配 | 中 | 去掉动态部分，用 `contains(@attr,'稳定部分')` 匹配 |
| 6 | 完整路径（/html/body/...） | 低 | 从根节点到目标元素的完整路径 |
| 7 | AI action | AI | 自动定位，每次消耗 AI token |

**录制生成结果说明**：

| 录制结果 | 含义 | 是否需要手动调整 |
|---------|------|:---:|
| 优先级 1-3 的 selector | 稳定可靠 | 通常不需要 |
| 优先级 4-6 的 selector | 可能不够稳定 | 建议检查，必要时添加 fallback |
| `action: "ai"` | 无法自动提取稳定 selector | 可保留或手动优化为确定性步骤 |

### iframe 选择器自动提取

录制时遇到 iframe 会自动提取选择器，优先级如下：

| 阶段 | 策略 | 示例输出 |
|------|------|---------|
| 1 | 单属性精确 | `//iframe[@id='x']`, `//iframe[@name='x']`, `//iframe[contains(@src,'/path')]` |
| 2 | 多属性组合 | `//iframe[@name='x' and @title='y']` |
| 3 | 祖先路径（属性锚点） | `//*[@id='dialog']//iframe` |
| 4 | 属性包含匹配 | `//iframe[contains(@id,'stable')]` |
| 5 | 兜底 | `xpath=(//iframe)[1]` |

iframe 选择器同样会跳过动态值（如 `ext-gen1234`），优先使用稳定的属性。

嵌套 iframe 会沿 `ParentFrame` 链向上提取每层 selector，组成根→叶 selector 数组（如 `["//iframe[@id='main']", "//iframe[@name='inner']"]`），每层在其父文档内检查唯一性。

### iframe 链的变量支持（`{{}}`）

iframe selector 链的**每一层**都支持 `{{rowIndex}}`/`{{字段名}}`/`{{变量}}` 占位符，回放时自动替换（与普通 selector 同款）：

- **`{{rowIndex}}`**：loop phase 内录制时**自动参数化**（如行级 iframe 含行号时，录制产出 `"//iframe[@data-row='{{rowIndex}}']"`）；
- **`{{字段名}}`/`{{变量}}`**：录制不自动参数化，需人工修改脚本填入（iframe 框架定位一般不含业务字段值——字段值属于元素 selector，不属于 iframe 框架定位，故 fieldName 通常不进 iframe 链）。

### selector 脆弱度分级与录制期求助（2c 阶段）

录制期代码对 selector 做脆弱度评估 + 唯一性校验，高脆弱/多匹配/找不到时主动求助用户，避免脆弱 selector 静默落盘导致回放失败。

**priority 脆弱度分级**（ExtractAsync 提取时算 + AssessFragility 对 AI 手写/直接传入的 selector 评估）：

| 级别 | priority | 来源 | 求助？ |
|---|---|---|---|
| 稳定 | 1-6 | id/name/data-*/文本/语义/祖先组合锚点（稳定值） | ❌ |
| 中等 | 7 | contains（属性/文本包含） | ❌（自动 onError=ai-fallback-stop） |
| 高脆弱 | 8 | 纯路径/完整路径/无稳定锚点/动态值锚点（GUID 等） | ✅ 求助 |
| 阻断 | 9 | 7 阶段提取均失败 | ✅ 求助（保存 ai action） |

**求助选项矩阵**（代码触发 request_help 时统一）：
- 公共 5 选项：(a) 重新定位 / (b) AI 分析 HTML（**必填粘贴 HTML**，不粘前端拦截/后端返必填提示）/ (c) 手写 selector / (d) ai 节点（录制期执行）/ (e) 跳过
- 脆弱独有 (f)：使用脆弱的——仅 priority 8 脆弱 selector / iframe 链含脆弱层（位置选择器层/GUID id/动态锚点） 求助时出现（接受当前 selector + 自动 onError=ai-fallback-stop 兜底）
- 代码 HTML 独有 (l)：用代码已提取的 HTML--仅 priority 9（7阶段全失败）/ priority 8 脆弱 selector / iframe 脆弱层 求助时出现（代码已抓元素 outerHTML/iframe ctx 全量喂 AI，AI 据此手写稳定 selector；不想手粘 HTML 时选 (l)，与 (b) 互补）。count==0/多匹配/iframe 多 frame/chain null 不显示 (l)（提取不到/歧义，防 silent）
- 多匹配 / 找不到：无 (f)（必须解决唯一性）

**acceptFragile**（仅录制期 AI 工具参数，**不落盘脚本**）：用户选 (f) 后下轮 operate/save_step 传 `acceptFragile=true` + **同 selector** → 代码跳过脆弱检测（只查唯一性 count==1）+ 自动设 onError=ai-fallback-stop 落盘，该元素不再求助。**仅对刚求助过的同 selector 传；换 selector 必须不传**。每个 priority 8 / iframe 高脆弱都求助（防同元素循环纯靠 acceptFragile）。⑥：phase.Condition/LoopCondition priority>=8 脆弱也走 (f) 求助 + acceptFragile（对齐 step detect ④，修 #2 silent）。

**acceptAsIs**（仅录制期 AI 工具参数，**不落盘脚本**，ND7）：用户选 (h) 后下轮 operate/save_step 传 `acceptAsIs=true` + **同 selector/iframe** → 代码跳过存在性校验（count==0 找不到：录制期元素动态加载未到/跨域 iframe 访问不到/not 预期不可见）直接落盘 + onError=ai-fallback-stop 兜底。**仅对刚求助过的同值传；换值必须不传**。多匹配（count>1）不适用（回放必失败，不传 acceptAsIs）。⑥：phase.Condition/LoopCondition count==0 也走 (h) 求助 + acceptAsIs（对齐 step detect ②，修 #1 不可达）。

**iframeRef**（仅录制期 AI 工具参数，**不落盘脚本**，结论14）：iframe 脆弱/提取失败/多 frame 求助时用户选 (i)，或 AI 主动用 aria-ref 指认 iframe → operate/save_step/detect/fallback 传 `iframeRef=<aria-ref>`（指向 `<iframe>` 元素的全局 ref）→ 代码用 aria-ref 定位该 iframe，提取其到 MainFrame 的 selector 链（`ResolveIframeFromRefAsync`，不复用 FindFrameForTargetAsync——ND3 实证语义错位）。落盘为 `iframe:[链]`，iframeRef 清空不落盘。

**useLast**（多匹配取最后）：当前**仅 click 消费**（录制 click + 回放 click 都 `useLast?Last:First`）；fill/select/hover/pressKey/scroll 两端都不读 useLast（回放 Playwright strict mode 多匹配抛错）。未来 multiMatch 三态通用。

**iframe 链脆弱+唯一**（与 element 完全对齐）：每层唯一 + 脆弱层（位置选择器层/GUID id/动态锚点，`(//iframe)[N]` 无 name/src 锚点的高脆弱）求助 + 使用脆弱的 (f)；提取失败/某层不唯一也求助。

**全路径对齐（D10）**：所有 selector/iframe 入口（operate/save_step/detect/captcha 5 selector/fallback）走同一套校验关卡（唯一性 + 脆弱度 + acceptFragile 两模式 + priority>=7 自动 onError + 求助矩阵 a-f）。**⑥（2026-07-09）**：phase.Condition/LoopCondition 也纳入全路径对齐——EnsurePhaseAsync 用 `lenient=false` 反推 + 完整求助通道对齐 step detect（① 多 frame picker (j) / ② count==0 → (h) acceptAsIs / ③ strict 校验失败阻断 / ④ priority>=8 → (f) acceptFragile），复用 step 级同一套 helper（RequestMultiFrameHelpAsync/BuildHelpQuestion）+ `FragileType.DetectSelector` 集合 + acceptAsIs/acceptFragile 总开关（集合策略最终决策：不隔离 phase、不去总开关——软确认文字回复唯一通道靠总开关）。修 #1（count-based count==0 返 pce=null 致 (h) 不可达）/ #2（旧弃用 priority 出参致 silent）/ #3（系统性）。

### 录制时字段的省略与必传

录制时（AI 驱动录制引擎）对 step 各字段的填写规则如下，AI 按此规则省略冗余字段、只传必要值：

| 字段 | 录制填写规则 | 原因 |
|------|------|------|
| `name` | **必传** | 录制端 tool schema required + script-v2.json stepNode required name；同 phase 内唯一，供 goto 的 toStep 引用 |
| `description` | **必传**（录制 tool required） | 操作说明，驱动确定性执行；ai action 时作为指令、fallback 时提供上下文。注：此处「必传」指**录制 tool schema required**（operate/save_step 均含 description）；手写脚本时**普通步骤**为可选（script-v2.json stepNode required 仅 kind/action/name，见 §十一步骤修饰字段汇总「建议填写」），但 **ai action 步骤的 description 仍必传**（schema if/then：action=ai → required description） |
| `selector` | 录制**自动生成**（XPath，见上方优先级表），AI 无需手写 | 录制引擎按优先级从 DOM 提取；`forceAiAction` 步骤省略 |
| `fieldName` / `fieldLabel` | 字段注册时**成对必传** | 触发字段定义自动注册（其余 `fieldXxx` 见下） |
| `fieldType`/`fieldUiComponent`/`fieldRequired`/`fieldOptions` 等 | 任务描述未明确时**省略** | 录制引擎从 DOM 自动提取字段元数据，AI 仅在任务描述明确时覆盖（见 §四字段定义） |
| `onError`/`retry`/`timeout`/`maxAiTurns`/`condition`/`preSetup`/`fallback` 等修饰字段 | 任务描述未明确时**省略** | 回放用五级配置链默认值（见 §字段配置原则） |

> **省略原则总纲**：任务描述未提及的字段一律省略，避免显式写入默认值（冗余且掩盖真实配置来源）。代码默认值与配置链兜底见 §字段配置原则。

## 七、step action 类型详解

每个 step 通过 `action` 字段指定操作类型，共 22 种 action。

### step 基本结构

```json
{
  "kind": "step",
  "action": "操作类型",
  "selector": "XPath 选择器（录制生成；手工编辑时也可用 CSS）",
  "value": "填写/选择的值",
  "description": "操作描述",
  ...
}
```

### 基础交互（10 种）

#### navigate — 导航到 URL

| 字段 | 必填 | 说明 |
|------|:----:|------|
| `url` | **是** | 目标 URL，支持 `{{}}` 占位符 |
| `iframe` | 否 | iframe selector 链 `string[]`（根→叶，可手写覆盖，一般由录制自动生成；详见[第十一节 step 共享字段](#iframe-selector-链-string)） |
| `timeout` | 否 | 超时（毫秒） |

```json
{ "kind": "step", "action": "navigate", "url": "https://example.com/login" }
```

#### click — 点击/双击元素

| 字段 | 必填 | 说明 |
|------|:----:|------|
| `selector` | **是** | 目标元素选择器 |
| `doubleClick` | 否 | true 时双击，默认 false |
| `useLast` | 否 | 多匹配时取最后一个，默认取第一个 |
| `iframe` | 否 | iframe selector 链 `string[]`（根→叶，可手写覆盖，一般由录制自动生成） |
| `timeout` | 否 | 超时（毫秒） |
| `preSetup` | 否 | 执行前设置（如 dialogHandler） |

```json
{ "kind": "step", "action": "click", "selector": "//*[@id='submitBtn']" }
{ "kind": "step", "action": "click", "selector": "//*[@type='submit']", "useLast": true, "doubleClick": true }
```

#### fill — 清空后输入

| 字段 | 必填 | 说明 |
|------|:----:|------|
| `selector` | **是** | 输入框选择器 |
| `value` | **是** | 要输入的值，支持 `{{}}` |
| `pressEnter` | 否 | 输入后按回车 |
| `iframe` | 否 | iframe selector 链 `string[]`（根→叶，可手写覆盖，一般由录制自动生成） |
| `timeout` | 否 | 超时（毫秒） |
| `preSetup` | 否 | 执行前设置 |

```json
{ "kind": "step", "action": "fill", "selector": "//*[@id='name']", "value": "{{F_HTMC}}", "pressEnter": true }
```

#### type — 追加输入（不清空）

与 `fill` 参数相同，区别是追加输入而不是先清空。

```json
{ "kind": "step", "action": "type", "selector": "//*[@id='remark']", "value": "（续）" }
```

#### select — 下拉选择

| 字段 | 必填 | 说明 |
|------|:----:|------|
| `selector` | **是** | 下拉框选择器 |
| `value` | **是** | 选择的值 |
| `matchBy` | 否 | 匹配方式：`"value"`（默认，按 value 属性）/ `"label"`（按可见文本）/ `"index"`（按索引） |
| `iframe` | 否 | iframe selector 链 `string[]`（根→叶，可手写覆盖，一般由录制自动生成） |
| `timeout` | 否 | 超时（毫秒） |

```json
// 按 value 属性匹配（默认）
{ "kind": "step", "action": "select", "selector": "//*[@id='tax-rate']", "value": "13%" }

// 按可见文本匹配
{ "kind": "step", "action": "select", "selector": "//*[@id='contract-type']", "value": "劳务分包合同", "matchBy": "label" }

// 按索引匹配（0-based）
{ "kind": "step", "action": "select", "selector": "//*[@id='tax-rate']", "value": "2", "matchBy": "index" }
```

> **注意**：自定义 select 组件（layui/antd 等非原生 `<select>` 元素）无法用此 action，需用 `ai` action。

#### hover — 鼠标悬停

| 字段 | 必填 | 说明 |
|------|:----:|------|
| `selector` | **是** | 目标元素选择器 |
| `iframe` | 否 | iframe selector 链 `string[]`（根→叶，可手写覆盖，一般由录制自动生成） |

```json
{ "kind": "step", "action": "hover", "selector": "//*[@id='menu-item']" }
```

#### pressKey — 按键

| 字段 | 必填 | 说明 |
|------|:----:|------|
| `key` | **是** | 键值：`Enter`, `Tab`, `Escape`, `ArrowDown`, `Control+a`, `Shift+Tab` 等 |
| `selector` | 否 | 在指定元素上按键。省略则在页面级别按键 |

```json
{ "kind": "step", "action": "pressKey", "key": "Enter" }
{ "kind": "step", "action": "pressKey", "key": "Control+a", "selector": "//*[@id='input']" }
```

#### scroll — 滚动

| 字段 | 必填 | 说明 |
|------|:----:|------|
| `selector` | 条件 | 滚动到元素可见时使用（与 `direction` 二选一） |
| `direction` | 条件 | 滚动方向：`up` / `down` / `left` / `right` / `bottom` / `top`（与 `selector` 二选一） |
| `amount` | 否 | 滚动量（像素），direction 为 up/down/left/right 时使用 |

> **二选一约束**（批次9）：`selector` 与 `direction` 严格互斥——同时指定或都不指定都会校验报错（ScriptLoader 加载期 / schema / 回放 / 录制多层拦截）。

```json
// 滚动到元素可见
{ "kind": "step", "action": "scroll", "selector": "//*[@id='submit-btn']" }

// 向下滚动 300 像素
{ "kind": "step", "action": "scroll", "direction": "down", "amount": 300 }

// 滚动到页面底部
{ "kind": "step", "action": "scroll", "direction": "bottom" }
```

#### evaluate — 执行 JavaScript

| 字段 | 必填 | 说明 |
|------|:----:|------|
| `code` | **是** | JS 代码，支持 `{{}}` 占位符 |
| `args` | 否 | JS 参数列表，字符串元素支持 `{{}}` |
| `storeAs` | 否 | 将返回值存为变量 |
| `iframe` | 否 | iframe selector 链 `string[]`（根→叶，可手写覆盖，一般由录制自动生成） |

```json
{ "kind": "step", "action": "evaluate", "code": "document.querySelector('#hidden-field').value = '{{value}}'" }
{ "kind": "step", "action": "evaluate", "code": "({{a}} + {{b}}).toString()", "storeAs": "sum" }
```

> ⚠️ **`code` 中 `{{}}` 的使用边界**：`{{x}}` 是**纯文本拼接**——值经 `ToString()` 后直接拼进 JS 源码，无类型感知、无转义。仅**纯数字标量**可安全使用；字符串/布尔/对象等会出错或被注入。

| 值类型 | `code` 里 `{{x}}` | 说明 |
|---|---|---|
| 纯数字（`rowIndex` / 计数器 / `number` 字段） | ✅ 安全 | `ToString()` = `"100"`，合法 JS 数字字面量 |
| 浮点 | ✅ 安全 | `"1.5"` 同上 |
| 字符串 | ❌ | 裸拼当标识符报错；加引号后值含 `"`/`\`/换行仍破坏字面量 |
| 布尔 | ❌ | C# `ToString()` = `"True"`/`"False"`（大写），JS 需 `true`/`false` |
| `null` / 未设置 | ❌ | `null`→空串致 `var x = ;`；未设置原样保留 `{{x}}` 致非法标识符 |
| 对象 / 数组 | ❌ | `ToString()` = 类型名 |

**推荐**：字符串/布尔/对象/数组/外部输入值，用 `args` 注入（类型保留、无转义/注入、`null` 安全），`code` 里用 `arguments[n]` 取；`{{}}` 仅用于纯数字标量（如 `{{rowIndex}}`）。

```json
// ✅ 推荐：字符串/对象走 args，纯数字（rowIndex）用 {{}}
{ "kind": "step", "action": "evaluate",
  "args": ["{{userName}}", "{{detailtable}}"],
  "code": "(function(){var name=arguments[0], dt=arguments[1]; return dt[{{rowIndex}}];})()" }
// userName/dt 类型保留；rowIndex 数字安全（仅 loop 内有效）

// ❌ 避免：字符串裸拼（userName="张三" → var n = 张三; → ReferenceError）
{ "kind": "step", "action": "evaluate", "code": "var n = {{userName}};" }
```

> 注：`args` 纯变量引用（如 `["{{detailtable}}"]`、`["{{detailtable.id}}"]`）透传**原始对象**（数组/number，非字符串），支持 pluck 取字段列数组（`{{detailtable.id}}` → id 列）。老脚本若依赖字符串 `typeof === "string"` 判定，请显式 `String(...)` 包一层。`{{rowIndex}}` 仅在 loop phase 内有效（引擎按迭代设入 vars）。

#### captcha — 验证码识别

| 字段 | 必填 | 说明 |
|------|:----:|------|
| `captchaType` | **是** | 验证码类型：`text` / `slide` / `pixel` / `click` |
| `imageSelector` | 条件 | 验证码图片元素选择器（text/click 用） |
| `inputSelector` | 条件 | 输入框选择器（text 用） |
| `sliderSelector` | 条件 | 滑块按钮选择器（slide/pixel 用） |
| `targetSelector` | 条件 | **语义随 captchaType 不同**：slide=滑块小图（缺口拼图）；pixel=**不带缺口的完整背景图**；click=提示文字元素。详见下方契约说明 |
| `backgroundSelector` | 条件 | **语义随 captchaType 不同**：slide=背景大图（带缺口）；pixel=**带缺口的完整背景图**。详见下方契约说明 |
| `iframe` | 否 | iframe selector 链 `string[]`（根→叶，可手写覆盖，一般由录制自动生成） |

**各 captchaType 需要的字段**：

| 字段 | text | slide | pixel | click |
|------|:----:|:-----:|:-----:|:-----:|
| captchaType | **必填** | **必填** | **必填** | **必填** |
| imageSelector | **必填** | — | — | **必填** |
| inputSelector | **必填** | — | — | — |
| sliderSelector | — | **必填** | **必填** | — |
| targetSelector | — | **必填** | **必填** | **必填** |
| backgroundSelector | — | **必填** | **必填** | — |

```json
// 文字验证码
{
  "kind": "step", "action": "captcha",
  "captchaType": "text",
  "imageSelector": "//img[contains(@src,'captcha')]",
  "inputSelector": "//*[@name='captchaInput']"
}

// 拼图型滑块
{
  "kind": "step", "action": "captcha",
  "captchaType": "slide",
  "sliderSelector": "//*[contains(@class,'slider-btn')]",
  "targetSelector": "//img[contains(@class,'slider-piece')]",
  "backgroundSelector": "//img[contains(@class,'slider-bg')]"
}

// 像素差异型滑块
{
  "kind": "step", "action": "captcha",
  "captchaType": "pixel",
  "sliderSelector": "//*[contains(@class,'slider-btn')]",
  "targetSelector": "//img[contains(@class,'slider-before')]",
  "backgroundSelector": "//img[contains(@class,'slider-after')]"
}

// 点选型验证码
{
  "kind": "step", "action": "captcha",
  "captchaType": "click",
  "imageSelector": "//img[contains(@class,'captcha-img')]",
  "targetSelector": "//*[contains(@class,'target-words')]"
}
```

**slide vs pixel 的 targetSelector/backgroundSelector 语义区别（重要，易混淆）**：

两种类型都用 `targetSelector` + `backgroundSelector`，但**语义和调用的 ddddocr 端点完全不同**，脚本作者切勿把 pixel 的 `targetSelector` 误配成滑块小图：

| 类型 | targetSelector | backgroundSelector | ddddocr 端点 | 算法 |
|------|---------------|--------------------|-------------|------|
| `slide` | **小滑块图**（缺口拼图块，独立小图） | **背景大图**（带缺口的完整背景） | `/slide_match`（slide_match） | 小图在背景中匹配定位缺口位移 |
| `pixel` | **不带缺口的完整背景图**（遮挡前快照） | **带缺口的完整背景图**（遮挡后快照，两图同尺寸） | `/slide_comparison`（slide_comparison） | 两张完整图求像素差异位移 |

> captcha 服务为自写 `captcha_service/mini_server.py`（FastAPI + ddddocr 1.6.1，端口 8000，4 端点 `/ocr` `/det` `/slide_match` `/slide_comparison`，JSON+base64，顶层 `result` 信封）。文字型用 `/ocr`、点选型用 `/det`（detection+classification 返回每个目标 `{x,y,text}`）。

> **pixel 配置口诀**：`targetSelector` 指**无缺口**的完整背景，`backgroundSelector` 指**有缺口**的完整背景——两者是同一场景的遮挡前/后两张完整图，求差异位移；**不是**小滑块 + 背景大图。误配成 slide 语义会导致 ddddocr `comparison` 比对失败。

> captcha action 确定性处理（0 token），失败后自动 retry，retry 耗尽走 onError 策略（可触发 AI 视觉兜底）。
> **旋转验证码**不支持 captcha action，请用 `ai` action。

## 八、check 动作详解

`check` 是步骤级条件检测。语义：**条件为 true → 执行 then；条件为 false → 继续下一步**。

### 基本格式

```json
{ "kind": "step", "action": "check", "detect": { ... }, "then": "继续动作" }
```

### detect 表达式

#### 单条件格式

```json
{ "type": "url_changed" }
{ "type": "selector_visible", "selector": "//*[contains(@class,'error')]" }
{ "type": "page_contains", "keywords": ["成功"] }
{ "type": "dialog_contains", "keywords": ["錯誤", "失敗"] }
```

#### 组合条件

```json
// AND（所有条件都满足）
{ "all": [
  { "type": "selector_visible", "selector": "//*[contains(@class,'error-panel')]" },
  { "type": "page_contains", "keywords": ["失敗"] }
]}

// OR（任一条件满足）
{ "any": [
  { "type": "dialog_contains", "keywords": ["錯誤"] },
  { "type": "page_contains", "keywords": ["失敗"] }
]}

// NOT（条件取反）
{ "not": { "type": "page_contains", "keywords": ["成功"] } }

// 混合：(A OR B) AND NOT C
{ "all": [
  { "any": [
    { "type": "dialog_contains", "keywords": ["錯誤"] },
    { "type": "page_contains", "keywords": ["失敗"] }
  ]},
  { "not": { "type": "page_contains", "keywords": ["成功"] }}
]}
```

### 20 种 detect 类型

> **`iframe` 参数（selector 链）**：表中 detect 的 `iframe` 列均为 **`string[]` selector 链（根→叶）**，限定 detect 在哪个 iframe 文档内查找；`null`/省略=继承上层 `phase.iframe`，`[]`=强制主文档，有内容=指定链（D3 三态语义，与 step.iframe 一致）。

| detect type | 必填参数 | 选填参数 | 说明 |
|-------------|---------|---------|------|
| `url_changed` | — | — | URL 发生变化（与上一步 URL 对比） |
| `url_contains` | `value` | — | URL 包含指定字符串 |
| `selector_visible` | `selector` | `iframe` | 元素可见 |
| `selector_exists` | `selector` | `iframe` | 元素存在（不要求可见） |
| `selector_gone` | `selector` | `iframe` | 元素消失 |
| `selector_enabled` | `selector` | `iframe` | 元素可用（非 disabled） |
| `iframe_exists` | `selector` | `iframe` | iframe 存在。检查 `selector` 定位的 iframe 元素是否存在；`iframe`（选填）提供父上下文链（如检查 outer 内的 nested iframe） |
| `page_exists` | — | `urlContains` | 页面存在（新标签页或新窗口） |
| `dialog_contains` | `keywords` | — | 对话框内容包含关键词（数组） |
| `page_contains` | `keywords` | `iframe` | 页面文本包含关键词（任一匹配） |
| `selector_value` | `selector`, `value` | `iframe` | 元素 value 等于指定值 |
| `selector_text` | `selector`, `keywords` | `iframe` | 元素文本包含关键词（任一匹配） |
| `selector_checked` | `selector` | `iframe` | 复选框/单选框是否选中 |
| `selector_selected` | `selector`, `value` | `iframe` | 下拉框选中项的值 |
| `selector_count` | `selector`, `count` | `iframe` | 匹配元素数量等于指定值 |
| `new_row_appears` | `selector`（**必填**，DC9） | `iframe` | 表格出现新行（对比前后行数；selector 不可省，不可用默认 `table tbody tr`——grid 场景 silent 数 0） |
| `js` | `check` | — | JS 表达式判断（服务端执行，可访问 `vars` 和 `fillData`，**不能访问页面 DOM**） |
| `data_exists` | `field` | — | fillData 中指定字段存在且非空 |
| `document_ready` | `iframe` | — | 目标 frame 的 `document.readyState === 'complete'`。**`iframe` 可省略**（三态：`null`/省略=继承 phase.iframe；`[]`=主页面 / `[链]`=指定 iframe，决策2-A）；不取 selector。等 iframe/页面整体加载完成，比 `selector_visible`（仅代表单个元素可见）更可靠。可用于 `wait.until` / `check` detect / `step.condition` / `phase.condition` |
| `always` | — | — | 条件恒成立（用于链式 check 最后一个无条件分支） |

> **提示**：需要基于页面内容判断时，先用 `extract` 提取到 vars，再用 `js` detect 判断。

> **`iframe_exists`（方案 A）用法**：删去旧版 `frameId`，改用 `selector`（必填，要检查的 iframe 元素）+ 可选 `iframe`（父上下文 selector 链）。

```json
// 检查主文档下的 popup iframe 是否存在
{ "type": "iframe_exists", "selector": "//iframe[@id='popup']" }

// 检查 outer iframe 内嵌套的 nested iframe 是否存在
{ "type": "iframe_exists", "selector": "//iframe[@id='nested']", "iframe": ["//iframe[@id='outer']"] }
```

> **提示**：需要基于页面内容判断时，先用 `extract` 提取到 vars，再用 `js` detect 判断。

### then 值

| then 值 | 含义 | 适用场景 |
|---------|------|---------|
| `nothing` | 通过/no-op，继续下一步（原 continue） | 所有 phase |
| `continue` | 下一行，loop 下一次迭代（原 next） | **仅 loop phase** |
| `step_error` | check 检测到业务错误 → 按本 step `onError` 处理 | 所有 phase |
| `phase_success` | phase 成功结束 | 所有 phase |
| `phase_error` | 触发 phase onError（原 phase_fail，可选 `cleanupSteps` + `message`） | 所有 phase |
| `phase_rerun` | 重跑整个 phase（受 `settings.rerun.maxPhaseRerunCount` 限制） | 所有 phase |
| `row_rerun` | 重跑当前行（受 `settings.rerun.maxRowRerunCount` 限制） | **仅 loop phase** |
| `script_success` | 脚本成功结束 | 所有 phase |
| `script_fail` | 脚本失败终止（可选 `cleanupSteps` + `message`） | 所有 phase |
| `goto` | 跳转到指定 phase（`toPhase`）或 step（`toStep`） | 所有 phase |
| `break` | 退出循环 | **仅 loop phase** |

### check 附加字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `message` | string | `phase_error` / `script_fail` 时的错误消息 |
| `toPhase` | string | `goto` 时的目标 phase name |
| `toStep` | string | `goto` 时的目标 step name |
| `cleanupSteps` | StepNode[] | `phase_error` / `script_fail` 时执行的清理步骤 |

### check 完整示例

```json
// 登录后检查 URL 变化
{ "kind": "step", "action": "check", "detect": { "type": "url_changed" }, "then": "continue" }

// 检查登录错误
{ "kind": "step", "action": "check",
  "detect": { "type": "selector_visible", "selector": "//*[contains(@class,'login-error')]" },
  "then": "phase_error", "message": "登录失败" }

// 检查对话框含错误关键词 → 失败并清理
{ "kind": "step", "action": "check",
  "detect": { "type": "dialog_contains", "keywords": ["庫存", "找不到", "無效"] },
  "then": "phase_error", "cleanupSteps": [
    { "action": "click", "selector": "//*[normalize-space(.)='確認']" },
    { "action": "click", "selector": "//*[normalize-space(.)='放棄本賬單']" },
    { "action": "handleDialog", "accept": true }
  ]}

// 检查对话框含格式错误 → 重试
{ "kind": "step", "action": "check",
  "detect": { "type": "dialog_contains", "keywords": ["格式", "為空"] },
  "then": "row_rerun" }

// 检测业务错误（提交后校验失败）→ step_error 按本 step onError 处理（只走 stop/skip/ai-fallback-stop/ai-fallback-skip，不做 retry/fallback）
{ "kind": "step", "action": "check",
  "detect": { "type": "dialog_contains", "keywords": ["必填项不能为空", "格式错误"] },
  "then": "step_error", "onError": "ai-fallback-stop" }

// 提取单据编号后检查成功
{ "kind": "step", "action": "check",
  "detect": { "type": "js", "check": "vars.billCode !== null" },
  "then": "phase_success" }

// 检查附件字段存在才执行上传 phase
"condition": { "type": "data_exists", "field": "orderAttachments" }

// 循环中检查新行出现 → 继续下一行
{ "kind": "step", "action": "check",
  "detect": { "type": "new_row_appears", "selector": "#data-table tbody tr" },
  "then": "continue" }

// 无条件跳转
{ "kind": "step", "action": "check",
  "detect": { "type": "always" },
  "then": "goto", "toPhase": "save" }
```

## 九、wait 等待详解

`wait` 有两种用法：

### 固定时间等待

```json
{ "kind": "step", "action": "wait", "ms": 2000 }
```

### 条件等待（轮询直到满足或超时）

`until` 复用 detect 类型：

```json
// 等待加载动画消失
{ "kind": "step", "action": "wait",
  "until": { "type": "selector_gone", "selector": "//*[contains(@class,'loading-spinner')]" },
  "timeout": 10000 }

// 等待结果出现
{ "kind": "step", "action": "wait",
  "until": { "type": "selector_visible", "selector": "//*[@id='result']" },
  "timeout": 5000 }

// 自定义轮询间隔
{ "kind": "step", "action": "wait",
  "until": { "type": "selector_visible", "selector": "//*[@id='result']" },
  "pollInterval": 200, "timeout": 5000 }

// 等 iframe/页面整体加载完成（readyState === 'complete'）—— 比 selector_visible（仅代表单个元素可见）更可靠
// 主页面加载完成（iframe = [] 显式主文档）
{ "kind": "step", "action": "wait",
  "until": { "type": "document_ready", "iframe": [] },
  "timeout": 60000 }

// 指定 iframe 链加载完成（弹窗 iframe 异步加载场景，如等乙方选择器 iframe 加载完再 fill）
{ "kind": "step", "action": "wait",
  "until": { "type": "document_ready", "iframe": ["iframe[src*='SupplierSelectorIndex']"] },
  "timeout": 60000 }
```

> **`document_ready` 特殊规则**：`iframe` **可省略**（决策2-A：=继承 phase.iframe，常规留空即可）；评估特定 iframe 时传 `[]`=主页面 / `[链]`=指定 iframe）；不取 selector，评估目标 frame 的 `document.readyState === 'complete'`。**性能提示**：document_ready 单次评估最长等 30s（Playwright Locator.EvaluateAsync auto-wait 等 frame 出现），与 `pollInterval`（默认 500ms）轮询语义有冲突——配 iframe 链时 `timeout` 建议留足（≥60s），否则 wait timeout 内可能只够 1-2 轮评估。

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `ms` | int | 条件 | 固定等待时间（毫秒），与 `until` 二选一 |
| `until` | object | 条件 | 等待条件（直接写 detect 对象），与 `ms` 二选一 |
| `timeout` | int | 否 | 条件等待超时（毫秒），默认用 settings.defaultTimeout |
| `pollInterval` | int | 否 | 轮询间隔（毫秒），默认 500 |

## 十、变量机制（vars / storeAs）

### {{}} 占位符替换

步骤中支持 `{{fieldName}}` 占位符，运行时替换为实际值。

**支持 `{{}}` 的 step 字段**：

| 字段 | 适用步骤 |
|------|---------|
| `description` | 所有步骤 |
| `selector` | 所有使用 selector 的步骤 |
| `value` | fill / type / select / evaluate |
| `url` | navigate |
| `code` | evaluate |
| `filePath` | upload |
| `message` | check (phase_error/script_fail) |
| `dialogPromptText` | handleDialog |
| `keywords[]` | check (dialog_contains) |
| `iframe[]` | step/phase/detect/fallback 的 iframe selector 链（每层都替换） |

**detect 中也支持 `{{}}`**：selector、value、keywords[]、iframe[]。

**不需要替换的字段**：name、field、storeAs、toPhase、toStep、action、onError、then、matchBy 等标识符/枚举字段。

### 作用域栈

`{{}}` 替换从作用域栈逐层查找，内层覆盖外层：

```
作用域栈（从内到外）：
  当前 loop 的 rowData    → 最内层
  外层 loop 的 rowData    → ...
  fillData 顶层           → 最外层
  vars                    → 最后查找
```

**执行示例**（嵌套 loop）：

```
fill_travel_table (loop, loopSource: consumption)，第1轮
作用域栈: [consumption[0], EXPENSE_RECORDS[0], fillData]

{{consumeDate}}      → consumption[0] 找到
{{category}}         → consumption[0] 没有 → EXPENSE_RECORDS[0] 找到
{{department}}       → 前两层没有 → fillData 找到
```

### 对象属性访问（Route B）

当变量值为 Dictionary 对象时，可以用 `{{name.key}}` 语法逐层访问属性：
- `{{shot.path}}` → 从 vars["shot"] 对象中取 path 属性
- `{{shot.dataUrl}}` → 从 vars["shot"] 对象中取 dataUrl 属性
- `{{stats.total}}` → 从 vars["stats"] 对象中取 total 属性
- `{{a.b.c}}` → 多层嵌套取值

向后兼容：`{{varName}}`（无 `.`）行为完全不变。
属性不存在时保留原始占位符。

### storeAs 返回值

通过 `storeAs` 将步骤执行结果存入 vars，后续步骤用 `{{varName}}` 引用。

| action | storeAs 返回值 |
|--------|---------------|
| `extract` | 提取到的值（文本/属性/URL/计数等） |
| `evaluate` | JS 执行结果 |
| `navigate` | 最终 URL |
| `click` | 有 storeAs 时读取页面实际值 |
| `fill` | 有 storeAs 时读取页面实际值 |
| `type` | 有 storeAs 时读取页面实际值 |
| `select` | 选中值（value 字符串，仅单选；多选 storeAs 跳过+告警，不支持） |
| `screenshot` | 由 storeContent 决定：path(文件绝对路径) / dataUrl(base64 data URL) / both(含 path 和 dataUrl 的对象) |
| `ai` | AI done 时返回的 result 值 |
| `handleDialog` | dialog 消息文本 |
| `switchTab` | 切换后的页面 URL |
| 其他 | null |

**单变量和多变量格式**：

```json
// 单变量
{ "action": "extract", "selector": "//*[@id='billNo']", "property": "value", "storeAs": "billCode" }

// 多变量（AI 返回 JSON 对象时按 key 拆分存入）
{ "action": "ai", "storeAs": { "billCode": "单据编号", "totalAmount": "金额" } }
```

### 特殊变量

| 变量名 | 来源 | 说明 |
|--------|------|------|
| `{{rowIndex}}` | loop phase 引擎自动设置 | 当前迭代行索引（从 0 开始，加 rowIndexOffset） |
| `{{fieldName}}` | fillData 用户提供 | 填报数据直接引用 |
| 自定义变量 | step 的 `storeAs` | 通过 storeAs 存入的任意变量 |

## 十一、步骤修饰字段汇总

以下字段可添加到任何 step 上，用于控制执行行为：

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `name` | string | - | 步骤名称，**必传**（同 phase 内唯一，供 goto 的 toStep 引用；与 Phase/Field.name 统一，决策13/审计 #42 全链路必传——录制端 tool schema required name + script-v2.json stepNode required name） |
| `description` | string | - | 操作描述。AI fallback 时提供上下文，ai action 时作为指令。建议填写 |
| `field` | string | - | 关联的 field 名称。AI fallback 时自动提取字段元数据 |
| `iframe` | string[] | 继承 phase 级值 | **iframe selector 链（根→叶，每层一个 XPath 或 CSS selector）**，限定元素查找的 iframe 文档。`null`/省略=未配（继承 `phase.iframe`）；`[]`（显式空数组）=强制主文档（不继承，覆盖回主文档）；有内容=指定链。一般由录制自动反推生成，也可手写覆盖。示例见下方「iframe selector 链 `string[]`」。**（形态 A 前写法：`iframe` 是引用 `iframeConfig` id 的字符串，已废弃）** |
| `skipIfDataEmpty` | bool | false | fillData 中关联字段为 null/空串/空数组时跳过；值为 0/false **不跳过**（零值是有意义的选择） |
| `skipIfElementMissing` | bool | false | 页面元素不存在时跳过 |
| `condition` | object | - | 执行前条件检测（直接写 detect 对象），不满足则跳过此步 |
| `fallback` | object | - | 失败后备选方案（备选选择器或备选动作） |
| `preSetup` | object | - | 执行前设置（对话框处理、等 LoadState.Load） |
| `retry` | object | - | 步骤级重试配置 |
| `onError` | string | 继承上级 | 覆盖错误策略 |
| `timeout` | int | 继承 settings | 步骤执行超时（毫秒） |
| `maxAiTurns` | int | 30 | AI 最大执行轮次 |

### iframe selector 链 `string[]`

`step.iframe` 是 **string[]**（根→叶的 selector 链），把目标元素所在的 iframe 嵌套路径"固化"进脚本。录制时引擎按目标（ref/selector/keywords）反推所在 frame，沿 `ParentFrame` 链向上提取每层 selector 组成链，回放时按链逐层 `FrameLocator` 定位到目标 iframe 文档。

**D3 三态语义**（`step.iframe`/`phase.iframe`/`detect.iframe`/`fallback.iframe` 通用）：

| 取值 | 语义 |
|------|------|
| `null` / 省略 | **未配**：继承上层 `phase.iframe`（step）或保持主文档（phase/detect 无上层时） |
| `[]`（显式空数组） | **强制主文档**：不继承，覆盖回主文档（用于"本步不在 iframe 内，即使 phase.iframe 有值"） |
| 有内容（如 `["//iframe[@id='form']"]`） | **指定链**：按链根→叶定位 |

> **`document_ready` 说明**（决策2-A）：`document_ready` detect 的 `iframe` **可省略**（=继承 phase.iframe）——常规留空；评估特定 iframe 时传 `[]`=主页面 或 `[链]`=指定 iframe。与其他 detect/step/phase 的 `iframe` D3 三态一致。

**示例**：

```json
// 单层 iframe
{ "kind": "step", "action": "fill", "selector": "//*[@id='name']", "value": "x",
  "iframe": ["//iframe[@id='form']"] }

// 嵌套 iframe（主→outer→nested）
{ "kind": "step", "action": "fill", "selector": "//*[@id='deep']", "value": "y",
  "iframe": ["//iframe[@id='outer']", "//iframe[@name='nested']"] }

// [] = 强制主文档（不继承 phase.iframe）
{ "kind": "step", "action": "click", "selector": "//*[@id='top-bar']", "iframe": [] }

// 省略 = 继承 phase.iframe
{ "kind": "step", "action": "click", "selector": "//*[@id='submit']" }
```

> **变量支持**：链每层支持 `{{rowIndex}}`（loop 内录制自动参数化）/ `{{字段名}}`/`{{变量}}`，回放时替换。详见[第六节 iframe 链的变量支持](#iframe-链的变量支持)。

### 字段配置原则（任务描述驱动 + 配置兜底）

`onError` / `retry` / `timeout` / `maxAiTurns` / `doubleClick` 等「任务描述驱动 + 配置兜底」类字段，遵循优先级：
1. **任务描述明确要求**（如「失败则跳过」「最多尝试 3 次」「超时 60 秒」）→ 按描述配置具体值
2. **任务描述未提及** → **省略该字段**，回放用五级配置链默认值（appsettings → 脚本 settings → 引擎默认）

避免显式写入默认值（如 `"maxAiTurns": 30`）——冗余且掩盖真实配置来源。录制时同理：AI 默认不写这些字段，只在任务描述明确时才传。

### field 字段详解

`field` 显式关联 step 到 `fields[]` 中的字段定义：

```json
// 显式声明（推荐）
{ "action": "fill", "selector": "//*[@id='F_HTMC']", "value": "{{F_HTMC}}", "field": "F_HTMC" }

// 自动推断：value 中 {{F_HTMC}} 匹配到 fields[] 中的 F_HTMC
{ "action": "fill", "selector": "//*[@id='F_HTMC']", "value": "{{F_HTMC}}" }

// 无关联（点击按钮等）
{ "action": "click", "selector": "//*[@id='submitBtn']" }
```

### skipIfDataEmpty vs skipIfElementMissing

| 字段 | 检查对象 | 行为 |
|------|---------|------|
| `skipIfDataEmpty` | fillData 中的字段值 | null/空串/空数组 → 跳过；0/false → **不跳过** |
| `skipIfElementMissing` | 页面元素 | 元素不存在 → 跳过 |

### fallback（失败后备选方案）

```json
// 备选选择器
{ "action": "click", "selector": "//*[@id='submitBtn']", "fallback": { "selector": "//*[normalize-space(.)='提交']" } }

// 备选动作
{ "action": "fill", "selector": "//*[@id='date']", "value": "2026-01-01",
  "fallback": { "action": "evaluate", "code": "document.querySelector('#date').value = '2026-01-01'" } }

// 多级 fallback
{ "action": "click", "selector": "//*[@id='btn1']",
  "fallback": { "selector": "//*[@id='btn2']", "fallback": { "selector": "//*[normalize-space(.)='確認']" } } }
```

### preSetup（执行前设置）

```json
// 对话框自动处理
{ "action": "click", "selector": "//*[@id='save']",
  "preSetup": { "dialogHandler": "auto_accept" } }

// 仅匹配特定文本的对话框
{ "action": "click", "selector": "//*[@id='save']",
  "preSetup": { "dialogHandler": "auto_accept", "dialogText": "确认" } }
```

| preSetup 字段 | 值 | 说明 |
|--------|------|------|
| `dialogHandler` | `"auto_accept"` / `"auto_dismiss"` | 自动接受或关闭对话框 |
| `dialogText` | string | 可选，仅匹配包含此文本的对话框 |

### 导航后等待范式（wait step 替代 waitLoadAfterNavigation）

`waitLoadAfterNavigation` 已于 2026-07-14 彻底移除（navigate/goBack/reload 自带 `WaitUntil=load` 冗余；click 弹窗类 racy 掉弹窗有害）。click 触发整页跳转或弹窗后需补等时，用显式 `wait` step 替代：

- **click 触发整页跳转（等目标 URL）**：`{ "action": "wait", "until": { "type": "url_contains", "value": "/dashboard" } }`（对应 Playwright `WaitForURL`，等 URL 变化到目标片段）
- **click 触发整页跳转（等任意变化）**：`{ "action": "wait", "until": { "type": "url_changed" } }`（等 URL 发生变化）
- **click 弹窗/局部更新**：`{ "action": "wait", "until": { "type": "selector_visible", "selector": "//*[@id='menu']" }, "timeout": 10000 }`（等目标元素出现）
- **timeout 取值**：慢站点建议 15000-30000ms（默认复用 step.timeout / appsettings defaultTimeout）

引用 [DetectEvaluator.cs](SmartFilling.Engine/Engine/DetectEvaluator.cs) 的 `url_changed`/`url_contains` 实现 + 录制提示词已有"click 跳转验证范式"对齐。

## 十二、错误处理与容错机制

### 六层容错

执行顺序：步骤条件 → 步骤重试 → 失败回退 → 结果分支 → AI 兜底

| 层 | 时机 | 机制 |
|----|------|------|
| 1 | phase 执行前 | `condition`（跳过整个 phase） |
| 2 | step 执行前 | `condition` / `skipIfElementMissing` / `skipIfDataEmpty` |
| 3 | step 失败时 | `retry`（自动重试 N 次） |
| 4 | step 重试耗尽时 | `fallback`（确定性备选方案） |
| 5 | step 成功后 | `check`（条件检测 + 控制流） |
| 6 | 以上都无法处理 | step 级 + phase 级 AI 兜底（`ai-fallback-stop` / `ai-fallback-skip`） |

### onError 错误策略

通过 `onError` 字段控制失败后的行为，支持**五级配置链**：

```
step.onError → phase.onError → script.settings.onError → appsettings.onError → 模型默认值 "stop"
```

每级为 null 时继承上级。

| 值 | 行为 |
|----|------|
| `stop` | 立即失败，终止脚本（**默认值**） |
| `skip` | 跳过此步骤/phase，继续执行 |
| `ai-fallback-stop` | 先调用 AI 兜底，AI 也失败则终止 |
| `ai-fallback-skip` | 先调用 AI 兜底，AI 也失败则跳过 |

> **注意 1**：`ai` action 失败**不触发** step 级 AI fallback（ai action 本身已是 AI 交互），直接走 onError 策略。
>
> **注意 2**：ai phase 失败**不触发** phase 级 AI fallback（ai phase 自身已是 AI 自主执行模式），此时 `ai-fallback-stop` 降级为 `stop`，`ai-fallback-skip` 降级为 `skip`。
>
> **注意 3**：`onError` 与 check 的 `then` 是**两个独立字段，值域不交叉**。`onError` 只取 `stop`/`skip`/`ai-fallback-stop`/`ai-fallback-skip`；`then` 取 `nothing`/`continue`/`step_error`/`phase_success`/`phase_error`/`phase_rerun`/`row_rerun`/`script_success`/`script_fail`/`goto`/`break`。`script_fail`/`phase_error` 等是 `then` 专属值，**绝不要填 `onError`**（schema enum 会拒；想要"失败即终止脚本"语义用 `onError=stop`，不要因想要"失败"语义就填 `script_fail`）。

### AI fallback 最佳实践与可观测性（#6）

用 `ai-fallback-stop`/`ai-fallback-skip` 的 step/phase，以及 `ai` action / `ai` phase，配置时遵循以下实践，让 AI 兜底/自主执行更可靠、过程可观测：

**1. `description` 写清修复目标 + 失败场景 + 成功标志**

AI 兜底/ai action 的 `description` 是 AI 理解任务的核心（instruction 传 description + 任务数据 JSON，不展开后续步骤）。写清：
- **修复目标**：这一步要达成什么（如"重新定位并填写供应商编号"）。
- **可能失败场景**：常见失败原因（如"弹窗 iframe 未加载完""selector 失效"）。
- **成功标志**：怎样算成功（如"乙方字段填入了正确编号"）。

**2. `storeAs` 多变量用 Dict（object 格式）**

`ai` action / `extract` 等需返回多个变量时，`storeAs` 用 object 格式 `{ "变量名": "描述" }`（不是单 string）。下游用 `{{变量名}}` 或 `{{name.key}}` 引用。单变量才用 string。

**3. `storeAs` 的 key 集合与 `description` 让 AI 提取的变量对齐**

`storeAs` object 的每个 key，其含义应在 `description` 中描述（AI 据 description 决定提取什么）。若 key 在 description 中未提及，AI 可能漏提取该变量（silent，仅缺 key 告警兜底）。建议：description 明确列出要提取的每个值，与 storeAs key 一一对应。**人工核对**：录制期请人工确认 storeAs key 与 description 描述对齐（当前无录制期自动 warning，回放期仅缺 key 告警兜底）。

**4. `document_ready` 等加载完成**（见[第九节 wait](#九wait-等待详解)）

`wait.until` 用 `document_ready` 等 iframe/页面整体加载完成（readyState=complete），比 `selector_visible`（仅单个元素可见）更可靠。可省略 `until.iframe`（`[]`=主页面 / `[链]`=指定 iframe）。

**5. AI 操作过程前端可见（可观测性）**

AI 兜底/ai action/ai phase 执行时，每轮工具调用、成功/失败、token 消耗都会推送到前端日志区（带 🤖 前缀）。用 `ai-fallback-*` 的步骤失败时会显示 `❌ step 失败: ...`，AI 兜底成功显示 `✅ AI 兜底成功`。便于诊断 AI 行为，无需查 worker 文件日志。

### 任务完成上报与失败呈现（P3 完成上报架构收敛）

脚本执行结束后的**终态信号由 `completionHandler` 单一出口**按任务实际状态发出（HTTP/WS 两模式对称），根治"完成/停止竞争"与"失败伪装成停止"两类隐患（engine 内不再直接发完成事件，移交 `TaskExecutionService` 的 finally → completionHandler）：

| 任务实际状态 | 完成信号 | 前端语义 |
|------|------|------|
| 成功 | `TaskCompleted`（`result=Success`，携带 `resultData`=returnData 全量数据字典 + `stats` AI 统计） | 填报完成，展示 returnData |
| 用户取消 | `TaskStopped`（`reason`=取消原因） | 已停止（停止语义，非失败） |
| 确定性失败 / AI 兜底失败 / 超时 | `TaskCompleted`（`result=Failed`，携带 `errorMessage` + `failureType`） | 填报失败（**不伪装停止**） |

`failureType` 枚举（`Deterministic`/`AiFallback`/`Timeout`/`Cancelled`/`None`）前端 log 显示前缀（如 `[AiFallback]`、`[Deterministic]`）供运维诊断，UI 主体不做区分。失败路径同样回填 `returnData`（含已成功提取的数据 + 失败时的 `lastError`），供调用方诊断。`returnData` 全量 JSON 同时落 Serilog 日志（截断 4KB + 换行清洗），便于日志轮询验证。

### 录制保存与终态行为

录制（非填报）的保存/终态行为（取消/超时/AI 未回填 aiGoal 的不完整脚本如何处理）：

| 行为 | 说明 |
|------|------|
| 保存弹框触发 | 录制正常完成（Success）或取消/超时（Cancelled）且有已录步骤时，前端弹出保存框；**Failed 不弹**（Failed 脚本不完整不保存）。 |
| 校验失败可强制保存 | 保存走 `ScriptLoader.ValidateAndGetErrors` + `ValidateSchemaAndGetErrors` **两层（业务+schema）校验**（lenientNull 动态取 `RecordOutputAllFields`，与 `GetScript` 一致——录制含 null 的脚本不被 schema required 误杀），不完整脚本缺 `aiGoal` 等会校验失败 → 前端返 `400 + validationFailed=true` → 用户 `confirm` 后可带 `forceSave=true` 重发**跳过两层校验**直接存。强制保存的脚本**脚本列表标 ⚠️**提醒用户（业务或 schema 校验失败的脚本可能无法正常执行）。`HasErrors` 反映 schema 或业务任一层校验失败。 |
| 前置脚本带病 → 录制 Failed | 录制时选了前置脚本，前置脚本**带病**（schema 或业务校验失败，如缺 aiGoal、含未知字段 `ifrmae` 拼错）或**结构坏**（JSON 损坏）→ 抛 `PrerequisiteScriptFailedException` → 录制 **Failed** 如实上报（前端日志显示「前置脚本 X 校验失败」）。仅**文件不存在**的前置跳过（前置可选，用户选「无」直接开始属正常，非病态）。 |

> 强制保存脚本的主要问题是缺 `aiGoal`（业务校验范畴，`HasErrors` 能抓并标 ⚠️），但 Worker 填报执行时会明确报失败（HTTP 400 / WS FailTask），不会 silent。`forceSave` 跳过 schema+业务两层校验会一并放过「静默取错值」类硬校验（命名冲突等）+ schema 架构校验（含未知字段检测），属已知风险。

### 录制期请求帮助组件（运行时行为）

录制（非填报）时，AI 遇到不确定情况会通过 `request_help` 工具暂停询问用户（operate 失败时代码也会自动触发），前端弹出 `#interactive-help` 请求帮助组件等待回复：

| 行为 | 说明 |
|------|------|
| **等待回复（无独立超时）** | 请求帮助组件**无 5 分钟等独立超时**，持续等待用户回复直到录制总超时（appsettings `MaxScriptDuration`，默认 60 分钟）。用户不回复不会自动结束录制（保持等待）。 |
| **回复方式** | 在请求帮助组件输入框填回复 → 点发送（调 `POST /api/record/respond`）→ 唤醒阻塞的录制循环继续。 |
| **录制终态自动关闭** | 录制结束（完成 `Success` / 取消 / 超时 / 失败 任一终态）时，前端自动关闭 `#interactive-help` 组件并清空输入框（原 TaskCompleted handler 漏关，导致请求帮助组件挂着不关闭）。 |
| **取消/超时如实上报** | 取消/超时上报为 `TaskCompleted(result=Cancelled)` + 区分原因的 errorMessage（`录制超时(N分钟)` / `用户取消录制` / `录制达到最大轮次限制(N轮)`，N 取自 `MaxScriptDuration`/`MaxRecordTurns`），保留已录步骤弹出保存框可保存（不再伪装成"录制完成"）。 |

> 注：填报（Worker 执行脚本）模式不支持请求帮助（由确定性脚本驱动），请求帮助仅录制模式有。

### 对话框处理三种机制

| 机制 | 触发方式 | 适用场景 |
|------|---------|---------|
| `preSetup.dialogHandler` | 步骤执行前注册事件，对话框出现时立即处理 | 预期会弹出的对话框 |
| `action: "handleDialog"` | 作为独立步骤，主动等待对话框 | 需要主动处理的对话框 |
| 引擎级意外对话框兜底 | action 超时后自动检测并 accept | 未预期的阻塞对话框 |

## 十四、JSON Schema 与编辑器使用

v2 脚本提供完整的 JSON Schema 文件（`script-v2.json`），基于 JSON Schema Draft 2020-12 规范。在编辑器中引用 Schema 后，可获得智能补全、悬停提示和实时校验，大幅降低脚本编写门槛。

### 14.1 获取 Schema 文件

Schema 文件位于引擎类库中：

```
SmartFilling.Engine/Schema/script-v2.json
```

将此文件复制到你的脚本工作目录（与脚本 JSON 文件同级），或放到公共可访问的位置。

### 14.2 在脚本中引用 Schema

在脚本 JSON 文件的**第一行**添加 `"$schema"` 字段：

```json
{
  "$schema": "script-v2.json",
  "version": 2,
  "scriptId": "my-script",
  "name": "示例",
  ...
}
```

`$schema` 的值可以是：
- **相对路径**：`"script-v2.json"`（同目录下）
- **相对路径**：`"../schema/script-v2.json"`（上级目录）
- **远程 URL**：`"https://your-server.com/schema/script-v2.json"`

### 14.3 VS Code 使用方式

VS Code 原生支持 JSON Schema，无需安装插件。

**步骤**：

1. 确保 Schema 文件已复制到可访问路径
2. 在脚本文件头部添加 `"$schema"` 引用
3. 正常编辑脚本即可获得以下能力

**智能补全**：

在任意位置按 `Ctrl+Space`（或输入 `"` 后自动触发），编辑器会提示当前上下文下可用的字段名。例如：

- 在 phase 对象中，提示 `name`、`type`、`aiGoal`、`steps` 等字段
- 在 `action` 字段中，提示 22 种可选值（`fill`、`click`、`check` 等）
- 在 `onError` 字段中，提示 `stop`、`skip`、`ai-fallback-stop`、`ai-fallback-skip`
- 在 `type`（detect）字段中，提示 20 种检测类型

**悬停提示**：

鼠标悬停在任意字段上，显示中文说明。例如：
- 悬停 `onError` → "错误策略：stop=终止 / skip=跳过 / ai-fallback-stop=AI兜底后终止 / ai-fallback-skip=AI兜底后跳过"
- 悬停 `captchaType` → "验证码类型：text=文字识别 / slide=拼图滑块 / pixel=像素差异 / click=点选"

**实时校验**：

编辑时即时标出错误，常见校验包括：

| 错误类型 | 示例 | 表现 |
|---------|------|------|
| 缺少必填字段 | phase 没有 `name` | 红色波浪线 + 错误提示 |
| 枚举值拼写错误 | `"onError": "ai-fallback"` | 标红（正确值是 `ai-fallback-stop` 或 `ai-fallback-skip`） |
| 字段名拼写错误 | `"seletor": "..."` | 标红（正确拼写是 `selector`） |
| action 必填字段缺失 | `"action": "fill"` 但没有 `value` | 通过 if/then 条件校验标红 |
| detect 类型参数缺失 | `"type": "selector_visible"` 但没有 `selector` | 通过 if/then 条件校验标红 |
| 未知字段 | `"unknownField": "..."` | 标红（Schema 禁止未定义字段） |

**校验问题面板**：

按 `Ctrl+Shift+M` 打开问题面板（Problems），可查看当前脚本所有校验错误的列表，双击跳转到对应位置。

### 14.4 Schema 校验范围

Schema 覆盖脚本的全部核心结构：

| 校验内容 | 说明 |
|---------|------|
| 顶层结构 | `version`、`scriptId`、`name`、`phases` 等必填字段 |
| 11 种核心类型 | ScriptV2、PhaseNode、StepNode、DetectCondition、FieldDefinition 等 |
| 22 种 action 条件校验 | 根据 `action` 值自动约束必填字段（如 `fill` 必须有 `selector` + `value`） |
| 20 种 detect 类型校验 | 根据 `type` 值自动约束参数（如 `selector_visible` 必须有 `selector`） |
| 全部枚举值约束 | onError 4 种、then 11 种、field type 6 种、uiComponent 10 种等 |
| 禁止未定义字段 | `additionalProperties: false`，拼写错误不会被静默忽略 |
| ai phase 步骤校验 | type 为 `"ai"` 的 phase 中，steps 只允许 action 必传为 `"ai"`（不可省略）的 StepNode，不允许嵌套 PhaseNode |

### 14.5 其他编辑器支持

| 编辑器 | 支持方式 |
|--------|---------|
| **VS Code** | 原生支持，添加 `$schema` 即可 |
| **JetBrains IDE**（WebStorm/IntelliJ） | 原生支持 JSON Schema，添加 `$schema` 即可 |
| **其他编辑器** | 支持 JSON Schema Draft 2020-12 的编辑器均可使用 |

### 14.6 引擎运行时校验

除了编辑器校验，引擎在加载脚本时也会进行运行时校验（`ScriptLoader`），包括：

- JSON 格式校验（反序列化是否成功）
- **JSON Schema 校验**（嵌入 `script-v2.json`，防字段残留/类型错误；含 required 必填字段校验）
- 业务规则校验：
  - phase name 脚本内全局唯一
  - field name 脚本内全局唯一（含嵌套 fields）
  - step name 同一父 phase 的 steps 内唯一
  - 顶层 phases 只允许 phase 节点
  - `{{X}}` 占位符中的 X 是否在 fields 中有定义（防止引用不存在的字段，含 `step.iframe`/`phase.iframe`/`detect.iframe`/`fallback.iframe` selector 链内出现的 `{{}}`）

**宽松校验模式**：Schema 的 `detectCondition` 是 `oneOf`+`additionalProperties:false`、可选字段类型不接受 null。录制器在 `AppOptions.RecordOutputAllFields=true` 时会输出全部字段（含大量 null），导致严格 Schema 校验失败。此时脚本加载走宽松模式——校验前先用 `StripNulls` 递归移除 null 字段（null 与字段缺失语义等价，不改变脚本行为）。App 侧由 `RecordOutputAllFields` 控制、Worker 侧由 `WorkerOptions.IgnoreNullFieldsOnScriptValidation`（默认 true）控制。

运行时校验会在脚本加载时报错并给出明确的错误信息，编辑器校验 + 引擎校验双重保障脚本正确性。**填报时若脚本加载/校验失败**，前端会显示 `⚠️` 错误提示（而非静默显示"未知单据"），便于定位问题。

**反向代理部署（R2-8）**：Worker 的 SignalR Hub（`/hubs/fill`）URL 解析优先用 `WorkerOptions.BaseUrl`（显式配置前端可达地址，最可靠）。未配 `BaseUrl` 时用请求的 `Scheme://Host`，反向代理（Nginx 等）下 `Host` 会变成**内部地址**导致前端连不上。两种解法二选一：
1. **配置 `WorkerOptions.BaseUrl`** 为前端可达的 Worker 外部地址（推荐，最可靠）；
2. 确保反向代理转发 `X-Forwarded-Host` / `X-Forwarded-Proto` 头，并在 Worker 的 `ForwardedHeadersOptions` 配置 `KnownNetworks`/`KnownProxies`（跨机代理必需——Worker Program.cs 已启用 `UseForwardedHeaders`，但默认仅信任 loopback 代理，跨机代理的 `X-Forwarded-*` 会被忽略，必须显式登记代理 IP/网段）。
